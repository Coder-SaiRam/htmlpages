Core Concepts — Authentication vs Authorization
WHO are you? WHAT can you do?
🪪
Authentication — WHO are you?
identity verification
Verifying who the user is. User provides credentials. System validates them. On success, creates a SecurityContext with the user's Authentication object for the thread.
Methods: Username+Password · JWT Token · OAuth2 · SAML · API Key · Certificate
UsernamePasswordAuthenticationToken
AuthenticationManager
✈️ Airport check-in — show passport, prove you are who you claim
🚪
Authorization — WHAT can you do?
permission enforcement
After authentication, determines what the user is allowed to access. Checks roles/authorities against required permissions. Happens AFTER authentication — cannot authorize an unknown user.
Methods: Roles (ROLE_ADMIN) · Authorities (read:users) · ACL · SpEL expressions
AccessDecisionManager
GrantedAuthority
🎫 Boarding gate — identity proven at check-in, boarding pass grants access to specific flight
🧩
SecurityContext
Holds the authenticated user's Authentication object. Thread-local per request. Automatically cleared after response.
ThreadLocal
🎫
Authentication
Core interface containing: principal (who), credentials (password), authorities (roles), isAuthenticated flag.
principal + roles
👤
UserDetails
Interface your User entity implements. Provides username, password hash, authorities, and account status flags (locked, expired, enabled).
implements UserDetails
🔑
GrantedAuthority
A permission granted to a user. Roles are prefixed ROLE_. Fine-grained authorities like user:read are also common.
ROLE_ADMIN

SecurityConfig — Spring Boot 3.x / Security 6.x
the main configuration class
SecurityConfig.java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity          // enables @PreAuthorize, @PostAuthorize
public class SecurityConfig {

    private final JwtAuthFilter jwtAuthFilter;
    private final UserDetailsService userDetailsService;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())                          // disable for REST APIs
            .cors(cors -> cors.configurationSource(corsConfig()))   // allow frontend origin
            .sessionManagement(s -> s
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)) // no sessions - JWT
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()        // public: login, register
                .requestMatchers("/api/public/**").permitAll()      // public: health, docs
                .requestMatchers("/api/admin/**").hasRole("ADMIN")  // admin only
                .requestMatchers(HttpMethod.GET, "/api/products/**").hasAnyRole("USER","ADMIN")
                .anyRequest().authenticated()                       // everything else needs auth
            )
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint(new HttpStatusEntryPoint(HttpStatus.UNAUTHORIZED))
                .accessDeniedHandler(new BearerTokenAccessDeniedHandler())
            )
            .build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12);   // strength 12 — never store plain text!
    }

    @Bean
    public AuthenticationManager authenticationManager(
            AuthenticationConfiguration config) throws Exception {
        return config.getAuthenticationManager();
    }
}

Security Filter Chain — The Request Pipeline
every request passes through these in order
💡
Every HTTP request passes through the Security Filter Chain. Each filter can: allow the request to continue, reject it (throw exception), or modify it. Filters run in a fixed order — order matters! Your custom filters are inserted at specific positions.
Filter ClassWhat It DoesWhen Active
① DisableEncodeUrlFilter
Prevents session ID being appended to URLs (security risk). First filter, always runs.
Always
② SecurityContextHolder­Filter
Loads SecurityContext from repository at request start; saves it back at end. Replaces old SecurityContextPersistenceFilter in Security 6.
Always
③ HeaderWriterFilter
Adds security response headers: X-Content-Type-Options, X-Frame-Options (clickjacking), X-XSS-Protection, HSTS (HTTPS enforcement).
Always
④ CorsFilter
Handles CORS preflight OPTIONS requests early. Sets Access-Control-Allow-* headers. Must run before auth filters to allow preflight through.
CORS on
⑤ CsrfFilter
Validates CSRF token on state-changing requests (POST/PUT/DELETE/PATCH). Disabled for stateless REST APIs that use JWT — tokens are CSRF-safe.
Web apps
⑥ LogoutFilter
Intercepts logout URL (default /logout). Clears SecurityContext, invalidates HTTP session, deletes remember-me cookies. Fully configurable.
Session auth
⑦ YOUR JwtAuthFilter
Custom filter you write! Extracts JWT from Authorization header, validates signature + expiry, sets Authentication in SecurityContext.
JWT apps
⑧ UsernamePassword­AuthenticationFilter
Handles POST /login — extracts username+password, delegates to AuthenticationManager. On success calls successHandler; on fail calls failureHandler.
Form login
⑨ BasicAuthentication­Filter
Handles HTTP Basic auth — decodes Base64 from Authorization: Basic {base64(user:pass)} header. Simple but credentials sent every request.
Basic auth
⑩ BearerTokenAuthentication­Filter
Handles OAuth2 Bearer tokens. Validates against auth server introspection or local JWT keys. Used when Spring Security OAuth2 Resource Server is configured.
OAuth2
⑪ RequestCacheAware­Filter
After login, redirects user to the original URL they were trying to access before being redirected to login. Restores saved request from cache.
Session
⑫ ExceptionTranslation­Filter
Catches AuthenticationException → 401, and AccessDeniedException → 403. The bridge between Java exceptions and HTTP error responses.
Always
⑬ AuthorizationFilter
Final authorization check — evaluates the URL-based access rules from your authorizeHttpRequests() config. Replaces FilterSecurityInterceptor in Security 6.
Always

Writing a Custom Filter — JWT Auth Filter
the most common custom filter you'll write
JwtAuthFilter.java
@Component
@RequiredArgsConstructor
public class JwtAuthFilter extends OncePerRequestFilter {

    private final JwtService jwtService;
    private final UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain filterChain) throws ServletException, IOException {

        // 1. Extract Authorization header
        final String authHeader = request.getHeader("Authorization");

        // 2. Skip if no Bearer token present
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response);  // pass to next filter
            return;
        }

        // 3. Extract JWT from header
        final String jwt = authHeader.substring(7);   // "Bearer " is 7 chars
        final String username = jwtService.extractUsername(jwt);

        // 4. Only authenticate if user not already authenticated in context
        if (username != null && SecurityContextHolder.getContext().getAuthentication() == null) {
            UserDetails userDetails = userDetailsService.loadUserByUsername(username);

            // 5. Validate token signature + expiry
            if (jwtService.isTokenValid(jwt, userDetails)) {
                UsernamePasswordAuthenticationToken authToken =
                    new UsernamePasswordAuthenticationToken(
                        userDetails, null, userDetails.getAuthorities()
                    );
                authToken.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));

                // 6. Set authentication in SecurityContext for this request
                SecurityContextHolder.getContext().setAuthentication(authToken);
            }
        }
        // 7. Always continue filter chain
        filterChain.doFilter(request, response);
    }
}

AuthController — Login Endpoint
AuthController.java
@RestController
@RequestMapping("/api/auth")
@RequiredArgsConstructor
public class AuthController {

    private final AuthenticationManager authManager;
    private final UserDetailsService userDetailsService;
    private final JwtService jwtService;
    private final PasswordEncoder passwordEncoder;
    private final UserRepository userRepository;

    @PostMapping("/login")
    public ResponseEntity<AuthResponse> login(
            @RequestBody LoginRequest request) {

        // This throws AuthenticationException if credentials are wrong
        authManager.authenticate(
            new UsernamePasswordAuthenticationToken(
                request.getUsername(),
                request.getPassword()
            )
        );

        // If we reach here, authentication succeeded
        UserDetails user = userDetailsService
            .loadUserByUsername(request.getUsername());

        String token = jwtService.generateToken(user);
        return ResponseEntity.ok(new AuthResponse(token));
    }

    @PostMapping("/register")
    public ResponseEntity<String> register(
            @RequestBody RegisterRequest request) {
        User user = User.builder()
            .username(request.getUsername())
            // ALWAYS encode before saving!
            .password(passwordEncoder.encode(request.getPassword()))
            .roles(List.of("ROLE_USER"))
            .build();
        userRepository.save(user);
        return ResponseEntity.ok("User registered successfully");
    }
}

UserDetailsService Implementation
CustomUserDetailsService.java
@Service
@RequiredArgsConstructor
public class CustomUserDetailsService implements UserDetailsService {

    private final UserRepository userRepository;

    @Override
    public UserDetails loadUserByUsername(String username)
            throws UsernameNotFoundException {
        // Load from YOUR database
        User user = userRepository.findByUsername(username)
            .orElseThrow(() ->
                new UsernameNotFoundException("User not found: " + username)
            );

        // Convert to Spring Security's UserDetails
        return org.springframework.security.core.userdetails.User
            .withUsername(user.getUsername())
            .password(user.getPassword())       // already BCrypt hashed
            .authorities(user.getRoles().stream()
                .map(SimpleGrantedAuthority::new)
                .collect(Collectors.toList()))
            .accountExpired(!user.isActive())
            .credentialsExpired(false)
            .disabled(!user.isEnabled())
            .build();
    }
}

// Your JPA User entity
@Entity
@Data
@Builder
public class User {
    @Id @GeneratedValue
    private Long id;
    @Column(unique = true)
    private String username;
    private String password;           // BCrypt hash stored here
    @ElementCollection(fetch = EAGER)
    private List<String> roles;       // ["ROLE_USER", "ROLE_ADMIN"]
    private boolean enabled = true;
    private boolean active = true;
}

URL-Based Authorization
in SecurityFilterChain
SecurityConfig.java — authorizeHttpRequests
.authorizeHttpRequests(auth -> auth
    // Public - no auth needed
    .requestMatchers("/api/auth/**").permitAll()
    .requestMatchers("/api/public/**").permitAll()
    .requestMatchers("/actuator/health").permitAll()

    // Role-based access
    .requestMatchers("/api/admin/**").hasRole("ADMIN")
    .requestMatchers("/api/manager/**").hasAnyRole("ADMIN", "MANAGER")

    // Authority-based (fine-grained)
    .requestMatchers(HttpMethod.DELETE, "/api/**")
        .hasAuthority("permission:delete")

    // Authenticated users only (any role)
    .requestMatchers("/api/user/**").authenticated()

    // SpEL expression for complex rules
    .requestMatchers("/api/reports/**")
        .access(new WebExpressionAuthorizationManager(
            "hasRole('ADMIN') or (hasRole('MANAGER') and @reportService.canAccess(authentication))"
        ))

    // Catch-all — deny everything else
    .anyRequest().authenticated()
)
📊
Role vs Authority
ConceptExampleUse When
RoleROLE_ADMIN, ROLE_USERBroad, coarse-grained access groups
Authorityuser:read, order:deleteFine-grained per-action permissions
hasRole()hasRole("ADMIN")Auto-prepends ROLE_ prefix
hasAuthority()hasAuthority("ROLE_ADMIN")Exact string match, no prefix
hasAnyRole()hasAnyRole("A","B")Multiple allowed roles
⚠️
Order matters! Rules are evaluated top-to-bottom. More specific patterns must come BEFORE .anyRequest(). Put permitAll() rules first.

🔴 Header (red)
{ "alg": "HS256", "typ": "JWT" }
Algorithm used to sign the token. Base64Url encoded.
🟣 Payload / Claims (purple)
{ "sub": "user123", "roles": ["ROLE_USER"], "iat": 1700000000, "exp": 1700086400 }
NOT encrypted — anyone can read it. Never put passwords or sensitive data here!
🟢 Signature (teal)
HMACSHA256(base64(header) + "." + base64(payload), secretKey)
Proves the token wasn't tampered with. Only server can produce/verify with secret key.

JwtService — Complete Implementation
JwtService.java
@Service
public class JwtService {

    // Store secret in application.properties, not in code!
    @Value("${jwt.secret}")
    private String secretKey;

    @Value("${jwt.expiration:86400000}") // 24 hours default
    private long jwtExpiration;

    // Generate token for a user
    public String generateToken(UserDetails userDetails) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("roles", userDetails.getAuthorities().stream()
            .map(GrantedAuthority::getAuthority)
            .collect(Collectors.toList()));
        return buildToken(claims, userDetails.getUsername());
    }

    private String buildToken(Map<String, Object> claims, String subject) {
        return Jwts.builder()
            .claims(claims)
            .subject(subject)
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + jwtExpiration))
            .signWith(getSigningKey())
            .compact();
    }

    // Validate token
    public boolean isTokenValid(String token, UserDetails userDetails) {
        final String username = extractUsername(token);
        return username.equals(userDetails.getUsername())
            && !isTokenExpired(token);
    }

    public String extractUsername(String token) {
        return extractClaim(token, Claims::getSubject);
    }

    private boolean isTokenExpired(String token) {
        return extractExpiration(token).before(new Date());
    }

    private Date extractExpiration(String token) {
        return extractClaim(token, Claims::getExpiration);
    }

    public <T> T extractClaim(String token, Function<Claims, T> resolver) {
        return resolver.apply(extractAllClaims(token));
    }

    private Claims extractAllClaims(String token) {
        return Jwts.parser()
            .verifyWith(getSigningKey())
            .build()
            .parseSignedClaims(token)
            .getPayload();
    }

    private SecretKey getSigningKey() {
        byte[] keyBytes = Decoders.BASE64.decode(secretKey);
        return Keys.hmacShaKeyFor(keyBytes);
    }
}
⚠️
JWT Security Best Practices: Use HS256+ or RS256. Set short expiry (15min–24h). Use Refresh Tokens for long sessions. Store in HttpOnly cookies (not localStorage — XSS risk!). Rotate signing keys regularly. Never put sensitive data in payload — it's only Base64, not encrypted.
🔁
Refresh Token Pattern
keep sessions alive securely
Access tokens should be short-lived (15–60 min). Refresh tokens are long-lived (7–30 days) and stored in DB. When access token expires, client uses refresh token to get a new access token without re-login.
RefreshToken flow
@PostMapping("/refresh")
public ResponseEntity<AuthResponse> refreshToken(
        @RequestBody RefreshTokenRequest request) {

    return refreshTokenService
        .findByToken(request.getRefreshToken())
        .map(refreshTokenService::verifyExpiration)  // throws if expired
        .map(RefreshToken::getUser)
        .map(user -> {
            String accessToken = jwtService.generateToken(user);
            return ResponseEntity.ok(new AuthResponse(accessToken));
        })
        .orElseThrow(() -> new RuntimeException("Invalid refresh token"));
}

Custom OAuth2 User Service
CustomOAuth2UserService.java
@Service
@RequiredArgsConstructor
public class CustomOAuth2UserService
        extends DefaultOAuth2UserService {

    private final UserRepository userRepository;

    @Override
    public OAuth2User loadUser(OAuth2UserRequest request)
            throws OAuth2AuthenticationException {

        OAuth2User oAuth2User = super.loadUser(request);
        String provider = request.getClientRegistration()
            .getRegistrationId();                     // "google" or "github"

        String email = oAuth2User.getAttribute("email");
        String name  = oAuth2User.getAttribute("name");

        // Load or create user in your DB
        User user = userRepository.findByEmail(email)
            .orElseGet(() -> User.builder()
                .email(email)
                .name(name)
                .provider(provider)
                .roles(List.of("ROLE_USER"))
                .build());
        userRepository.save(user);

        // Return enriched OAuth2User with your roles
        return new CustomOAuth2User(oAuth2User, user.getRoles());
    }
}

CSRF & CORS — Web Security Essentials
cross-origin attacks and access control
🎭
CSRF — Cross-Site Request Forgery
tricking your browser into sending requests
Attack: Evil site tricks your browser into making a request to your bank with your cookies. Browser auto-sends cookies — so bank thinks it's you. Defence: Include a secret CSRF token that evil site can't read (same-origin policy).
✅
REST APIs with JWT: Disable CSRF (csrf.disable()). JWT is sent in Authorization header, not cookies — evil sites can't access headers cross-origin. No CSRF risk.
⚠️
Web apps with sessions/cookies: Keep CSRF enabled! Spring auto-generates CSRF tokens. Thymeleaf auto-inserts them in forms. Axios/Fetch: read XSRF-TOKEN cookie and send as X-XSRF-TOKEN header.
CSRF config choices
// Option 1: Disable for stateless REST API (JWT)
.csrf(csrf -> csrf.disable())

// Option 2: CSRF with cookie (for SPA frontends)
.csrf(csrf -> csrf
    .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
    .csrfTokenRequestHandler(new CsrfTokenRequestAttributeHandler())
)

// Option 3: Ignore specific endpoints only
.csrf(csrf -> csrf
    .ignoringRequestMatchers("/api/webhooks/**")
)
🌐
CORS — Cross-Origin Resource Sharing
allowing your React/Angular frontend
Problem: Browser blocks requests from http://localhost:3000 (React) to http://localhost:8080 (Spring) — different origins (port counts!). Solution: Configure Spring to return CORS headers that tell the browser: "this origin is allowed."
CorsConfig.java — production setup
@Bean
public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration config = new CorsConfiguration();

    // Allowed origins - use env var in prod!
    config.setAllowedOrigins(List.of(
        "http://localhost:3000",         // local dev
        "https://app.yourcompany.com"    // production
    ));

    // Allowed HTTP methods
    config.setAllowedMethods(List.of(
        "GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS"
    ));

    // Allowed request headers
    config.setAllowedHeaders(List.of(
        "Authorization", "Content-Type",
        "X-Requested-With", "Accept"
    ));

    // Allow credentials (cookies, auth headers)
    config.setAllowCredentials(true);

    // Cache preflight response for 1 hour
    config.setMaxAge(3600L);

    UrlBasedCorsConfigurationSource source =
        new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/api/**", config);
    return source;
}

Method Security — @PreAuthorize & Friends

💡
Enable with @EnableMethodSecurity on your config class. Secures individual methods using AOP. More granular than URL-based rules — you can check IDs, ownership, and complex conditions.
🔒
@PreAuthorize — Before Method Runs
OrderService.java
@Service
public class OrderService {

    // Only ADMIN can delete orders
    @PreAuthorize("hasRole('ADMIN')")
    public void deleteOrder(Long orderId) { ... }

    // ADMIN or MANAGER can approve
    @PreAuthorize("hasAnyRole('ADMIN', 'MANAGER')")
    public void approveOrder(Long orderId) { ... }

    // User can only update their OWN orders
    // authentication.name = logged-in username
    @PreAuthorize("@orderRepository.findById(#orderId)" +
                  ".get().getOwnerUsername() == authentication.name" +
                  " or hasRole('ADMIN')")
    public void updateOrder(Long orderId, OrderDto dto) { ... }

    // Check custom bean method
    @PreAuthorize("@securityService.canAccessOrder(#orderId)")
    public Order getOrder(Long orderId) { ... }

            .hasRole("CUSTOMER")
        .requestMatchers("/admin/**").hasRole("ADMIN")
        .anyRequest().authenticated()

      .addFilterBefore(jwtFilter,
          UsernamePasswordAuthenticationFilter.class)
      .build();
  }

@Component
@RequiredArgsConstructor
public class OAuth2SuccessHandler
        extends SimpleUrlAuthenticationSuccessHandler {

  private final JwtService jwtService;
  private final UserRepository userRepository;

  @Override
  public void onAuthenticationSuccess(
      HttpServletRequest req, HttpServletResponse res,
      Authentication authentication) throws IOException {

    CustomOAuth2User oAuth2User =
        (CustomOAuth2User) authentication.getPrincipal();

    // Find or create user in our DB
    User user = userRepository
        .findByEmail(oAuth2User.getEmail())
        .orElseGet(() -> createUserFromOAuth2(oAuth2User));

    // Issue our own JWT
    String token = jwtService.generateToken(user);

    // Redirect frontend with token in fragment
    // Frontend reads token and stores it
    getRedirectStrategy().sendRedirect(req, res,
        "http://localhost:3000/oauth-callback#token=" + token);
  }
}

AuditAspect.java — log every record access
@Aspect @Component @RequiredArgsConstructor
public class SecurityAuditAspect {

  private final AuditLogRepository auditRepo;

  @AfterReturning(
    pointcut = "@annotation(Auditable)",
    returning = "result"
  )
  public void logAccess(JoinPoint jp, Object result) {
    Authentication auth = SecurityContextHolder
        .getContext().getAuthentication();

    AuditLog log = AuditLog.builder()
        .username(auth.getName())
        .action(jp.getSignature().getName())
        .resource(result.getClass().getSimpleName())
        .timestamp(Instant.now())
        .ipAddress(getClientIp())
        .build();

    auditRepo.save(log); // every record access saved
  }
}

Cheatsheet + Interview Q&A
everything you need to remember
⚡ Key Annotations
@EnableWebSecurityActivates Spring Security — put on @Configuration class
@EnableMethodSecurityEnables @PreAuthorize, @PostAuthorize, @PreFilter, @PostFilter
@PreAuthorize("...")Security check BEFORE method runs — uses SpEL expressions
@PostAuthorize("...")Security check AFTER — can inspect returnObject value
@Secured("ROLE_X")Simpler annotation — no SpEL, just role names
@AuthenticationPrincipalInject current UserDetails into controller method parameter
@WithMockUserTest annotation — mock a logged-in user in unit tests
@WithUserDetails("bob")Test with real UserDetails loaded by name
🎯 SecurityContext Access
getContext().getAuthentication()Get current Authentication anywhere in your code
.getName()Get the authenticated username (principal name)
.getAuthorities()Get list of roles/permissions granted to user
.isAuthenticated()Check if user is authenticated (not anonymous)
.getPrincipal()Get the UserDetails object (cast to your User type)
clearContext()Clear authentication — use on logout
setAuthentication(auth)Manually set authenticated user — used in JWT filter
MODE_INHERITABLETHREADLOCALStrategy to propagate SecurityContext to child threads
❓ Interview Q&A — Part 1
Filter Chain? Ordered pipeline of security filters every request passes through. Each filter has single responsibility. Custom filters inserted at specific positions with addFilterBefore/After.
Authentication vs Authorization? Authentication = who are you (verify identity). Authorization = what can you do (check permissions). Auth always happens before Authz.
Why BCrypt? Bcrypt is slow by design (cost factor). Includes salt automatically. Resistant to rainbow table attacks. Never store plain text or MD5/SHA1 passwords.
JWT vs Session? JWT is stateless (no DB lookup per request), scalable, good for microservices. Sessions are stateful (stored server-side), easier to invalidate, good for monoliths.
CSRF vs CORS? CSRF is an attack (forged requests using your cookies). CORS is a browser policy (blocks cross-origin requests). They're unrelated — CORS doesn't prevent CSRF.
❓ Interview Q&A — Part 2
SecurityContext scope? ThreadLocal by default — each request/thread has its own context. Cleared after each request. Use MODE_INHERITABLETHREADLOCAL for async code.
hasRole vs hasAuthority? hasRole("ADMIN") auto-prepends ROLE_ prefix. hasAuthority("ROLE_ADMIN") is exact match. Same result if you follow the ROLE_ convention.
How to test security? Use @WithMockUser for mock users, @WithUserDetails for real DB users, MockMvc with .with(user(...)) for controller tests. @WithAnonymousUser for anonymous access tests.
401 vs 403? 401 Unauthorized = not authenticated (no valid credentials). 403 Forbidden = authenticated but not authorized (wrong role). ExceptionTranslationFilter handles the mapping.
How to get current user? SecurityContextHolder.getContext().getAuthentication() anywhere. @AuthenticationPrincipal UserDetails user in controller params. @CurrentSecurityContext for full context.

@WebMvcTest(AccountController.class)
@Import(SecurityConfig.class)
class AccountControllerTest {

    @Autowired MockMvc mockMvc;

    @Test
    @WithAnonymousUser  // unauthenticated
    void getBalance_unauthenticated_returns401() throws Exception {
        mockMvc.perform(get("/accounts/1/balance"))
            .andExpect(status().isUnauthorized());
    }

    @Test
    @WithMockUser(username = "alice", roles = {"CUSTOMER"})
    void getBalance_asOwner_returns200() throws Exception {
        // Mocked user has username "alice"
        mockMvc.perform(get("/accounts/1/balance"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.balance").exists());
    }

    @Test
    @WithMockUser(username = "bob", roles = {"CUSTOMER"})
    void deleteAccount_asCustomer_returns403() throws Exception {
        // CUSTOMER role cannot delete — ADMIN only
        mockMvc.perform(delete("/admin/accounts/1"))
            .andExpect(status().isForbidden());
    }

    @Test
    @WithMockUser(roles = {"ADMIN"})
    void deleteAccount_asAdmin_returns200() throws Exception {
        mockMvc.perform(delete("/admin/accounts/1"))
            .andExpect(status().isOk());
    }

    @Test
    void login_withValidCredentials_returnsJwt() throws Exception {
        String body = """
            {"username": "alice", "password": "password123"}
            """;
        mockMvc.perform(post("/api/auth/login")
                .contentType(MediaType.APPLICATION_JSON)
                .content(body))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.token").isNotEmpty());
    }
}
