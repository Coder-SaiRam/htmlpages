Application Properties
application.properties & application.yml
Central configuration files that control every aspect of your Spring Boot application. Define database connections, server ports, logging levels, custom properties, and framework behavior without touching code.
⚡ When to Use
  • Configure database connections, server ports, context paths
  • Set logging levels for packages or specific classes
  • Define custom application properties accessible via @Value
  • Toggle features, enable/disable auto-configuration
🏢 Real Scenario — Multi-Environment Setup
  • Development uses H2 in-memory database on port 8080.
  • Production uses PostgreSQL with SSL, runs on port 443.
  • Use application-dev.properties and application-prod.properties.
  • Activate with spring.profiles.active=prod or -Dspring.profiles.active=prod

Environment Variables & Profiles
Runtime Configuration & Environment-Specific Settings
Externalize configuration to environment variables for security and flexibility. Use profiles to maintain separate configurations for development, staging, and production without code changes.
⚡ When to Use
  • Store sensitive data (passwords, API keys) outside codebase
  • Deploy same artifact across environments (dev, staging, prod)
  • Override properties at runtime without rebuilding
  • CI/CD pipelines inject environment-specific values
🏢 Real Scenario — Database Credentials
  • Never hardcode spring.datasource.password in properties files.
  • Use environment variable: ${DB_PASSWORD}.
  • Set via OS: export DB_PASSWORD=secret.
  • Or Docker: docker run -e DB_PASSWORD=secret

  • For Kubernetes/Docker: inject via ConfigMaps and Secrets

@ConfigurationProperties
Type-Safe Configuration Binding
Bind groups of related properties to strongly-typed POJOs. Better than scattered @Value annotations. Provides validation, IDE autocomplete, and refactoring safety for complex configuration hierarchies.
⚡ When to Use
  • Multiple related properties (database config, API settings, feature flags)
  • Need validation on configuration values
  • Hierarchical/nested configuration structures
  • Prefer compile-time safety over runtime @Value strings
🏢 Real Scenario — Third-Party API Config
  • Your app integrates with Stripe API.
  • Need: API key, secret, webhook URL, timeout, retry config.
  • Create StripeProperties class with @ConfigurationProperties(prefix="stripe").
  • All stripe.* properties auto-bind to type-safe fields

SpEL - Spring Expression Language
Dynamic Property Resolution & Runtime Evaluation
SpEL is a powerful expression language for querying and manipulating objects at runtime. Use in @Value, XML configs, security rules. Supports method invocation, mathematical operations, conditional logic, collection filtering.
⚡ SpEL Capabilities
  • Property access: #{user.name}
  • Method invocation: #{user.getName()}
  • Mathematical operations: #{10 * 2 + 5}
  • Conditional logic: #{age > 18 ? 'adult' : 'minor'}
  • Collection filtering: #{users.?[age > 18]}
  • System properties: #{systemProperties['user.home']}
🏢 Real Scenario — Dynamic Configuration
  • Set timeout based on environment: prod = 30s, dev = 5s.
  • Use SpEL: @Value("#{environment == 'prod' ? 30 : 5}").
  • Calculate derived values from properties.
  • Access system properties and environment variables dynamically
SpEL Basics
Java
@Component
public class SpELExamples {

    // ===== PROPERTY ACCESS =====
    @Value("#{systemProperties['user.home']}")
    private String userHome;

    @Value("#{systemProperties['java.version']}")
    private String javaVersion;

    // Access environment variable
    @Value("#{environment['PATH']}")
    private String path;

    // ===== MATHEMATICAL OPERATIONS =====
    @Value("#{10 * 2 + 5}")
    private int calculated;  // 25

    @Value("#{ T(java.lang.Math).random() * 100 }")
    private double random;

    // ===== CONDITIONAL LOGIC =====
    @Value("#{${server.port} == 8080 ? 'dev' : 'prod'}")
    private String environment;

    @Value("#{'${spring.profiles.active}' == 'prod' ? 30000 : 5000}")
    private int timeout;

    // ===== STRING OPERATIONS =====
    @Value("#{'Hello ' + '${app.name}'}")
    private String greeting;

    @Value("#{'${app.name}'.toUpperCase()}")
    private String upperName;

    // ===== BEAN REFERENCE =====
    @Value("#{@userService.getUserCount()}")
    private int userCount;

    @Value("#{@appConfig.timeout}")
    private Duration configTimeout;

    // ===== NULL SAFETY =====
    @Value("#{config.name ?: 'default'}")  // Elvis operator
    private String nameWithDefault;

    @Value("#{config.value?.length()}")  // Safe navigation
    private Integer length;
}
Advanced SpEL - Collections
Java
@Component
public class CollectionSpEL {

    // ===== COLLECTION FILTERING =====
    // Filter collection: ?[condition]
    @Value("#{users.?[age > 18]}")  // Adults only
    private List<User> adults;

    // First match: ^[condition]
    @Value("#{users.^[age > 18]}")
    private User firstAdult;

    // Last match: $[condition]
    @Value("#{users.$[age > 18]}")
    private User lastAdult;

    // ===== PROJECTION =====
    // Extract property: ![property]
    @Value("#{users.![name]}")  // List of names
    private List<String> names;

    // Combined filtering + projection
    @Value("#{users.?[age > 18].![name]}")
    private List<String> adultNames;
}

// Example usage in configuration
@Configuration
public class SpELConfig {

    @Bean
    @ConditionalOnExpression("#{environment['ENABLE_CACHE'] == 'true'}")
    public CacheManager cacheManager() {
        return new ConcurrentMapCacheManager();
    }

    @Bean
    public RestTemplate restTemplate(
            @Value("#{ T(java.time.Duration).ofSeconds(${api.timeout:30}) }")
            Duration timeout) {

        HttpComponentsClientHttpRequestFactory factory =
            new HttpComponentsClientHttpRequestFactory();
        factory.setConnectTimeout((int) timeout.toMillis());

        return new RestTemplate(factory);
    }
}

🔧 SpEL Operators

Operator Example Description
+, -, *, /, % #{10 + 5} Arithmetic
==, !=, <, >, <=, >= #{age > 18} Comparison
and, or, not, ! #{active and verified} Logical
?: #{name ?: 'default'} Elvis (null coalescing)
?. #{user?.name} Safe navigation
^[] #{list.^[condition]} First match in collection
$[] #{list.$[condition]} Last match in collection
?[] #{list.?[condition]} Filter collection
![] #{list.![property]} Project collection
T() #{T(Math).PI} Type reference
💡 SpEL Best Practices
  • Use #{} for SpEL, ${} for property placeholders
  • Combine both: #{'${prop.value}'.toUpperCase()}
  • Use ?: (Elvis) for default values instead of null checks
  • Use ?. (safe navigation) to avoid NPE on property access
  • Keep expressions simple — complex logic belongs in Java code

Spring Beans & IoC Container
The Heart of Spring Framework
Beans are objects managed by Spring's IoC (Inversion of Control) container. Spring creates, configures, wires dependencies, and manages the entire lifecycle. You declare what you need; Spring handles instantiation and dependency injection.
⚡ What is a Bean?
  • An object created and managed by Spring IoC container
  • Defined via annotations (@Component, @Service, @Bean) or XML
  • Container handles instantiation, dependency injection, lifecycle
  • Singleton by default — one shared instance per container
🏢 Real Scenario — Service Layer
  • You have UserService that depends on UserRepository.
  • Mark UserRepository with @Repository — Spring creates bean.
  • Mark UserService with @Service — Spring creates bean.
  • Inject repository into service via constructor — Spring wires automatically
Bean Definition & Injection
Java
// 1. Component Scanning - Auto-detected beans
@Repository  // Specialized @Component for data access
public class UserRepository {
    public User findById(Long id) { /* DB logic */ }
}

@Service  // Specialized @Component for business logic
public class UserService {
    private final UserRepository repository;

    // Constructor Injection (recommended)
    public UserService(UserRepository repository) {
        this.repository = repository;
    }

    public User getUser(Long id) {
        return repository.findById(id);
    }
}

// 2. Java Config - Explicit bean creation
@Configuration
public class AppConfig {

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}

// 3. Using beans
@RestController
public class UserController {

    private final UserService userService;
    private final RestTemplate restTemplate;

    // Spring injects both beans via constructor
    public UserController(UserService userService,
                          RestTemplate restTemplate) {
        this.userService = userService;
        this.restTemplate = restTemplate;
    }
}

Bean Scopes
Controlling Bean Lifecycle & Instance Management
Scopes define how many instances of a bean exist and how long they live. Singleton (one shared instance), Prototype (new instance every time), Request/Session (web-scoped). Choose based on statefulness and performance needs.
⚡ Available Scopes
  • singleton (default): One instance per Spring container — shared globally
  • prototype: New instance every time bean is requested
  • request: One instance per HTTP request (web apps only)
  • session: One instance per HTTP session (web apps only)
  • application: One instance per ServletContext
  • Custom scopes: Define your own lifecycle management
🏢 Real Scenario — Shopping Cart
  • A ShoppingCart bean holds user-specific items.
  • Can't be singleton — would be shared across all users!
  • Use @Scope("session") — each HTTP session gets its own cart.
  • Cart persists across requests for the same user, cleared on logout

Transaction Management
@Transactional & ACID Properties
Control database transactions declaratively. @Transactional ensures operations are atomic, isolated, and can rollback on exceptions. Critical for data consistency in multi-step database operations.
⚡ When to Use @Transactional
  • Multiple database operations that must succeed or fail together
  • Read-write operations requiring isolation from other transactions
  • Prevent dirty reads, phantom reads, non-repeatable reads
  • Automatic rollback on runtime exceptions
🏢 Real Scenario — Money Transfer
  • Transfer $100 from Account A to Account B.
  • Debit A by $100, credit B by $100 — both must succeed.
  • If credit fails, debit must rollback — maintain data integrity.
  • @Transactional ensures atomicity: all or nothing
Transaction Examples
Java
@Service
public class BankService {

    private final AccountRepository accountRepo;

    // Default: readOnly=false, propagation=REQUIRED
    @Transactional
    public void transferMoney(Long fromId, Long toId, BigDecimal amount) {
        Account from = accountRepo.findById(fromId)
            .orElseThrow(() -> new NotFoundException());
        Account to = accountRepo.findById(toId)
            .orElseThrow(() -> new NotFoundException());

        from.debit(amount);  // Step 1
        to.credit(amount);    // Step 2

        // If exception thrown here, BOTH operations rollback ✓
        if (amount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new IllegalArgumentException("Invalid amount");
        }
    }

    // Read-only transaction — optimizations, prevents writes
    @Transactional(readOnly = true)
    public Account getAccount(Long id) {
        return accountRepo.findById(id)
            .orElseThrow(() -> new NotFoundException());
    }

    // Custom rollback rules
    @Transactional(
        rollbackFor = Exception.class,  // Rollback on checked exceptions too
        noRollbackFor = NotFoundException.class  // Don't rollback for this
    )
    public void updateAccount(Account account) {
        accountRepo.save(account);
    }

    // Isolation levels
    @Transactional(isolation = Isolation.SERIALIZABLE)
    public void criticalOperation() {
        // Highest isolation — prevents all concurrency issues
    }

    // Propagation behavior
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void auditLog(String message) {
        // Always create new transaction — independent of parent
    }
}

Level Dirty Read Non-Repeatable Read Phantom Read Performance
READ_UNCOMMITTED ❌ Allowed ❌ Allowed ❌ Allowed ⚡ Fastest
READ_COMMITTED ✅ Prevented ❌ Allowed ❌ Allowed ⚡ Fast
REPEATABLE_READ ✅ Prevented ✅ Prevented ❌ Allowed 🐢 Moderate
SERIALIZABLE ✅ Prevented ✅ Prevented ✅ Prevented 🐢 Slowest

🔄 Propagation Modes

Mode Behavior
REQUIRED (default) Use existing transaction or create new one
REQUIRES_NEW Always create new transaction, suspend existing
NESTED Execute within nested transaction (savepoint)
MANDATORY Must have existing transaction, else throw exception
SUPPORTS Use transaction if exists, else execute non-transactionally
NOT_SUPPORTED Execute non-transactionally, suspend existing transaction
NEVER Must NOT have transaction, else throw exception
⚠️ Common Mistakes
  • @Transactional only works on public methods
  • Calling transactional method from same class bypasses proxy — won't work
  • Default rollback: only on RuntimeException, not checked exceptions
  • Use readOnly=true for query methods — performance optimization
  • Long transactions lock resources — keep scope minimal

@RestControllerAdvice
public class GlobalExceptionHandler {

    // Handle validation errors
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ErrorResponse handleValidationErrors(
            MethodArgumentNotValidException ex) {

        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors()
            .forEach(error ->
                errors.put(error.getField(), error.getDefaultMessage())
            );

        return new ErrorResponse(
            "Validation failed",
            errors
        );
    }

    // Handle resource not found
    @ExceptionHandler(ResourceNotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public ErrorResponse handleNotFound(ResourceNotFoundException ex) {
        return new ErrorResponse(ex.getMessage());
    }

    // Handle duplicate resource
    @ExceptionHandler(DuplicateResourceException.class)
    @ResponseStatus(HttpStatus.CONFLICT)
    public ErrorResponse handleDuplicate(DuplicateResourceException ex) {
        return new ErrorResponse(ex.getMessage());
    }

    // Handle illegal arguments
    @ExceptionHandler(IllegalArgumentException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ErrorResponse handleBadRequest(IllegalArgumentException ex) {
        return new ErrorResponse(ex.getMessage());
    }

    // Catch-all for unexpected errors
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ErrorResponse handleGenericError(Exception ex) {
        // Log the exception
        // Don't expose internal details to client
        return new ErrorResponse("An unexpected error occurred");
    }
}

⚡ Versioning Strategies

JPA & Hibernate
Java Persistence API & ORM
JPA provides object-relational mapping to interact with databases using Java objects. Hibernate is the most popular JPA implementation. Define entities, relationships, and let JPA handle SQL generation and execution.
⚡ JPA Core Concepts
  • Entity: Java class mapped to database table
  • EntityManager: Interface to perform DB operations
  • JPQL: Object-oriented query language (queries on entities, not tables)
  • Relationships: OneToOne, OneToMany, ManyToOne, ManyToMany
  • Cascade: Propagate operations (save, delete) to related entities
  • Fetch: EAGER (load immediately) vs LAZY (load on access)
🏢 Real Scenario — Blog System
  • Blog has Users, Posts, Comments.
  • One User writes Many Posts (OneToMany).
  • One Post has Many Comments (OneToMany).
  • Define entities with relationships, JPA handles joins automatically

    @Transactional
    @Query("UPDATE User u SET u.active = :active WHERE u.id = :id")

    @Transactional

DTOs & Entity Mapping
Decoupling API from Database Layer
Use Data Transfer Objects (DTOs) for API contracts. Don't expose JPA entities directly — leads to tight coupling, security risks, and circular reference issues. Use MapStruct or ModelMapper for conversion.
⚡ Why Use DTOs?
  • Hide internal entity structure from clients
  • Prevent password/sensitive field exposure
  • Avoid circular reference (infinite recursion) in JSON serialization
  • API evolution independent of database schema
  • Validation specific to API layer, not persistence
🏢 Real Scenario — Circular Reference Problem
  • User has many Orders, Order has reference to User.
  • Serializing User entity serializes Orders, which serialize User again — infinite loop.
  • Solution: Use DTOs with only required fields, no circular refs.
  • MapStruct auto-generates mapping code at compile time

Request Processing Pipeline
Understanding Filters & Interceptors
Filters and interceptors intercept HTTP requests at different stages. Filters operate at servlet level (before Spring), interceptors at Spring MVC level (after routing). Both enable cross-cutting concerns like logging, authentication, auditing without modifying controllers.
⚡ Request Flow Architecture
  • Step 1: Request arrives at servlet container (Tomcat/Jetty)
  • Step 2: Filters execute (authentication, CORS, logging)
  • Step 3: DispatcherServlet receives request
  • Step 4: Interceptors preHandle() executes
  • Step 5: Controller method handles request
  • Step 6: Interceptors postHandle() executes
  • Step 7: View rendered (if applicable)
  • Step 8: Interceptors afterCompletion() executes
  • Step 9: Filters execute (response phase)
  • Step 10: Response sent to client
🏢 Real Scenario — Secure API with Logging
  • Filter 1 (Order 1): CORS - Allow frontend domain access.
  • Filter 2 (Order 2): JWT Authentication - Validate token, reject if invalid.
  • Filter 3 (Order 3): Request logging - Log method, URI, timestamp.
  • Interceptor 1: Audit logging - Log user ID, action, IP address.
  • Controller: Business logic execution.
  • Interceptor 2: Performance monitoring - Calculate execution time.
  • Response flows back through filters in reverse order

Servlet Filters
Early-Stage Request Processing
Filters operate at the servlet container level, intercepting requests before they reach Spring. Perfect for security, CORS, encoding, compression. Part of Java EE specification, work with any servlet-based framework.
⚡ When to Use Filters
  • Authentication & authorization - validate JWT tokens, session cookies
  • CORS configuration - handle preflight requests, set headers
  • Request/response logging - track all incoming traffic
  • Character encoding - set UTF-8 for requests/responses
  • Compression - gzip/deflate response bodies
  • Security headers - X-Frame-Options, CSP, HSTS
  • Rate limiting - throttle requests per IP/user
🏢 Real Scenario — JWT Authentication Filter
  • Every API request must include Authorization: Bearer <token> header.
  • Filter extracts token, validates signature, expiration.
  • If valid: extract user ID, set in SecurityContext, continue chain.
  • If invalid: return 401 Unauthorized, stop chain execution.
  • Runs before Spring processes request - early rejection saves resources
Filter Implementation Methods
Java
// ===== METHOD 1: @Component (Auto-Registration) =====
import javax.servlet.*;
import javax.servlet.http.*;
import org.springframework.core.annotation.Order;

@Component
@Order(1)  // Lower number = higher priority (executes earlier)
public class RequestLoggingFilter implements Filter {

    private static final Logger log = LoggerFactory.getLogger(RequestLoggingFilter.class);

    // Optional: called once during filter initialization
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        log.info("RequestLoggingFilter initialized");
    }

    // Main filter logic - called for every request
    @Override
    public void doFilter(ServletRequest request,
                       ServletResponse response,
                       FilterChain chain) throws IOException, ServletException {

        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse res = (HttpServletResponse) response;

        long startTime = System.currentTimeMillis();
        String requestId = UUID.randomUUID().toString();

        // Log request details
        log.info("[{}] Request: {} {} from {}",
                 requestId,
                 req.getMethod(),
                 req.getRequestURI(),
                 req.getRemoteAddr());

        // Add request ID to response header
        res.setHeader("X-Request-ID", requestId);

        // IMPORTANT: Continue the chain
        chain.doFilter(request, response);

        // After controller execution (response phase)
        long duration = System.currentTimeMillis() - startTime;
        log.info("[{}] Response: {} - {}ms",
                 requestId,
                 res.getStatus(),
                 duration);
    }

    // Optional: called once during filter destruction
    @Override
    public void destroy() {
        log.info("RequestLoggingFilter destroyed");
    }
}

@Component
@Order(2)
public class JwtAuthenticationFilter implements Filter {

    private static final String AUTHORIZATION_HEADER = "Authorization";
    private static final String BEARER_PREFIX = "Bearer ";

    private final JwtTokenProvider jwtTokenProvider;

    public JwtAuthenticationFilter(JwtTokenProvider jwtTokenProvider) {
        this.jwtTokenProvider = jwtTokenProvider;
    }

    @Override
    public void doFilter(ServletRequest request,
                       ServletResponse response,
                       FilterChain chain) throws IOException, ServletException {

        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse res = (HttpServletResponse) response;

        // Skip authentication for public endpoints
        if (isPublicEndpoint(req.getRequestURI())) {
            chain.doFilter(request, response);
            return;
        }

        try {
            // Extract token from header
            String token = extractToken(req);

            if (token == null) {
                sendUnauthorizedError(res, "Missing authorization token");
                return;
            }

            // Validate token
            if (!jwtTokenProvider.validateToken(token)) {
                sendUnauthorizedError(res, "Invalid or expired token");
                return;
            }

            // Extract user info from token
            String userId = jwtTokenProvider.getUserIdFromToken(token);
            List<String> roles = jwtTokenProvider.getRolesFromToken(token);

            // Set authentication in SecurityContext
            Authentication auth = new UsernamePasswordAuthenticationToken(
                userId, null, convertToAuthorities(roles)
            );
            SecurityContextHolder.getContext().setAuthentication(auth);

            // Continue to next filter/controller
            chain.doFilter(request, response);

        } catch (Exception e) {
            sendUnauthorizedError(res, "Authentication failed: " + e.getMessage());
        } finally {
            // Clear security context
            SecurityContextHolder.clearContext();
        }
    }

    private String extractToken(HttpServletRequest request) {
        String bearerToken = request.getHeader(AUTHORIZATION_HEADER);
        if (bearerToken != null && bearerToken.startsWith(BEARER_PREFIX)) {
            return bearerToken.substring(7);  // Remove "Bearer " prefix
        }
        return null;
    }

    private boolean isPublicEndpoint(String uri) {
        return uri.startsWith("/api/public") ||
               uri.equals("/api/login") ||
               uri.equals("/api/register");
    }

    private void sendUnauthorizedError(HttpServletResponse response, String message)
            throws IOException {
        response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
        response.setContentType("application/json");
        response.getWriter().write(String.format(
            "{\"error\": \"Unauthorized\", \"message\": \"%s\"}", message
        ));
    }
}

@Component
@Order(Ordered.HIGHEST_PRECEDENCE)
public class CorsFilter implements Filter {

    @Value("${app.cors.allowed-origins}")
    private String allowedOrigins;

    @Override
    public void doFilter(ServletRequest req,
                       ServletResponse res,
                       FilterChain chain) throws IOException, ServletException {

        HttpServletResponse response = (HttpServletResponse) res;
        HttpServletRequest request = (HttpServletRequest) req;

        // Set CORS headers
        response.setHeader("Access-Control-Allow-Origin", allowedOrigins);
        response.setHeader("Access-Control-Allow-Methods",
                              "GET, POST, PUT, DELETE, OPTIONS, PATCH");
        response.setHeader("Access-Control-Allow-Headers",
                              "Authorization, Content-Type, X-Requested-With");
        response.setHeader("Access-Control-Allow-Credentials", "true");
        response.setHeader("Access-Control-Max-Age", "3600");

        // Handle preflight OPTIONS request
        if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
            response.setStatus(HttpServletResponse.SC_OK);
            return;  // Don't continue chain for OPTIONS
        }

        chain.doFilter(req, res);
    }
}

  • Use @Order to control execution sequence - lower number runs first
  • Always call chain.doFilter() to continue, or explicitly send response to stop
  • CORS filter should have HIGHEST_PRECEDENCE to handle preflight first
  • Authentication filters should run early but after CORS
  • Logging filters can run at any point - consider both early and late logging
  • Use FilterRegistrationBean for filters that only apply to specific URLs
  • Be careful with @Component filters - they apply to ALL requests automatically

HandlerInterceptors
Spring MVC Request Interception
Interceptors operate within Spring MVC framework after handler mapping. Three-phase lifecycle (pre/post/after) gives fine-grained control. Full access to Spring beans, handler information, and ModelAndView. Perfect for auditing, logging, authorization checks.
⚡ When to Use Interceptors
  • Audit logging - track user actions with full context
  • Performance monitoring - measure controller execution time
  • Authorization checks - verify permissions based on controller annotations
  • Request preprocessing - modify request attributes before controller
  • Response postprocessing - modify ModelAndView after controller
  • Resource cleanup - close connections, clear thread locals
  • Metrics collection - gather business metrics per endpoint
🏢 Real Scenario — Audit Trail System
  • Track every user action: who, what, when, from where.
  • preHandle: Extract user ID from SecurityContext, log action start.
  • Controller executes: Business logic runs.
  • postHandle: Log success, capture return value.
  • afterCompletion: Log execution time, save audit record to database.
  • Even if exception occurs, afterCompletion still logs the attempt
Complete Interceptor Example
Java
import org.springframework.web.servlet.HandlerInterceptor;
import org.springframework.web.servlet.ModelAndView;
import org.springframework.web.method.HandlerMethod;

@Component
public class AuditInterceptor implements HandlerInterceptor {

    private static final Logger log = LoggerFactory.getLogger(AuditInterceptor.class);

    private final AuditService auditService;

    public AuditInterceptor(AuditService auditService) {
        this.auditService = auditService;  // DI works! ✓
    }

    /**
     * Phase 1: BEFORE controller execution
     * Return false to stop request processing
     * Return true to continue to controller
     */
    @Override
    public boolean preHandle(HttpServletRequest request,
                              HttpServletResponse response,
                              Object handler) throws Exception {

        long startTime = System.currentTimeMillis();
        request.setAttribute("startTime", startTime);

        // Extract handler info (only for @Controller methods)
        if (handler instanceof HandlerMethod) {
            HandlerMethod handlerMethod = (HandlerMethod) handler;

            // Get controller class and method
            Class<?> controllerClass = handlerMethod.getBeanType();
            Method method = handlerMethod.getMethod();

            // Check for custom annotation
            if (method.isAnnotationPresent(RequiresPermission.class)) {
                RequiresPermission permission = method.getAnnotation(RequiresPermission.class);
                if (!hasPermission(permission.value())) {
                    response.setStatus(HttpServletResponse.SC_FORBIDDEN);
                    return false;  // Stop execution
                }
            }

            log.info("[PRE] {} {} → {}.{}",
                     request.getMethod(),
                     request.getRequestURI(),
                     controllerClass.getSimpleName(),
                     method.getName());
        }

        // Get authenticated user
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth != null) {
            String username = auth.getName();
            request.setAttribute("currentUser", username);
            log.debug("User: {}", username);
        }

        return true;  // Continue to controller
    }

    /**
     * Phase 2: AFTER controller execution, BEFORE view rendering
     * Not called if exception thrown in controller
     * Can modify ModelAndView
     */
    @Override
    public void postHandle(HttpServletRequest request,
                            HttpServletResponse response,
                            Object handler,
                            ModelAndView modelAndView) throws Exception {

        log.info("[POST] Controller executed successfully");

        // Add custom attributes to model (for view rendering)
        if (modelAndView != null) {
            modelAndView.addObject("serverTime", LocalDateTime.now());
            modelAndView.addObject("requestId", request.getAttribute("requestId"));
        }

        // For REST APIs, modelAndView is typically null
    }

    /**
     * Phase 3: AFTER view rendering (or after postHandle if no view)
     * ALWAYS called, even if exception occurred
     * Perfect for cleanup and final logging
     */
    @Override
    public void afterCompletion(HttpServletRequest request,
                                 HttpServletResponse response,
                                 Object handler,
                                 Exception ex) throws Exception {

        Long startTime = (Long) request.getAttribute("startTime");
        long duration = System.currentTimeMillis() - startTime;

        String username = (String) request.getAttribute("currentUser");
        String uri = request.getRequestURI();
        int status = response.getStatus();

        // Log completion
        log.info("[COMPLETE] {} {} - Status: {} - Duration: {}ms",
                 request.getMethod(), uri, status, duration);

        // Save audit record
        AuditRecord audit = AuditRecord.builder()
            .username(username)
            .action(request.getMethod() + " " + uri)
            .ipAddress(request.getRemoteAddr())
            .statusCode(status)
            .duration(duration)
            .timestamp(LocalDateTime.now())
            .build();

        auditService.save(audit);

        // Log exception if present
        if (ex != null) {
            log.error("Exception during request processing: ", ex);
        }

        // Cleanup thread locals, close resources
        MDC.clear();
    }
}

Performance Monitoring Interceptor
Java
@Component
public class PerformanceInterceptor implements HandlerInterceptor {

    private static final Logger log = LoggerFactory.getLogger(PerformanceInterceptor.class);
    private static final long SLOW_REQUEST_THRESHOLD_MS = 1000;  // 1 second

    @Override
    public boolean preHandle(HttpServletRequest request,
                              HttpServletResponse response,
                              Object handler) {
        request.setAttribute("perfStartTime", System.nanoTime());
        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request,
                                 HttpServletResponse response,
                                 Object handler,
                                 Exception ex) {

        Long startTime = (Long) request.getAttribute("perfStartTime");
        long durationMs = (System.nanoTime() - startTime) / 1_000_000;

        String endpoint = request.getMethod() + " " + request.getRequestURI();

        // Log slow requests
        if (durationMs > SLOW_REQUEST_THRESHOLD_MS) {
            log.warn("⚠️ SLOW REQUEST: {} - {}ms", endpoint, durationMs);
        } else {
            log.debug("Performance: {} - {}ms", endpoint, durationMs);
        }

        // Send metrics to monitoring system (Prometheus, CloudWatch, etc.)
        Metrics.recordExecutionTime(endpoint, durationMs);
    }
}

🔄 Interceptor Lifecycle

Method When Called Return/Action Use Case
preHandle() Before controller Return false = stop, true = continue Authorization, validation, preprocessing
postHandle() After controller, before view Modify ModelAndView Add common model attributes, response modification
afterCompletion() After view rendering Always called (even on exception) Cleanup, logging, resource release, metrics
⚠️ Common Pitfalls
  • preHandle() return false stops execution — no postHandle/afterCompletion
  • postHandle() NOT called if controller throws exception
  • afterCompletion() ALWAYS called — use for mandatory cleanup
  • Interceptors only apply to controller-mapped requests (not static resources)
  • Order matters — lower order value runs first in preHandle, last in afterCompletion
  • Don't forget to register interceptors in WebMvcConfigurer
💡 Interceptor Best Practices
  • Use preHandle for authorization — fail fast before controller execution
  • Use afterCompletion for logging/metrics — always executes, handles exceptions
  • Store start time in request attributes in preHandle, read in afterCompletion
  • Access Spring beans via constructor injection — works perfectly
  • Use path patterns wisely — include/exclude for fine-grained control
  • Check handler type (instanceof HandlerMethod) before casting
  • Consider async requests — afterCompletion may be called on different thread

// Custom error handler
public class CustomResponseErrorHandler implements ResponseErrorHandler {

    @Override
    public boolean hasError(ClientHttpResponse response) throws IOException {
        return response.getStatusCode().is4xxClientError() ||
               response.getStatusCode().is5xxServerError();
    }

    @Override
    public void handleError(ClientHttpResponse response) throws IOException {
        if (response.getStatusCode().is4xxClientError()) {
            // 4xx errors
            if (response.getStatusCode() == HttpStatus.NOT_FOUND) {
                throw new ResourceNotFoundException("Resource not found");
            }
            throw new ClientErrorException("Client error: " + response.getStatusCode());
        }

        if (response.getStatusCode().is5xxServerError()) {
            // 5xx errors
            throw new ServerErrorException("Server error: " + response.getStatusCode());
        }
    }
}

⚡ When to Use WebClient
  • New Spring Boot projects (recommended over RestTemplate)
  • Reactive applications using Spring WebFlux
  • High-concurrency scenarios (many concurrent API calls)
  • Non-blocking I/O requirements
  • Streaming responses (Server-Sent Events)
  • Functional, fluent API preference
🏢 Real Scenario — Aggregating Multiple APIs
  • Dashboard needs data from User Service, Order Service, Analytics Service.
  • Sequential calls with RestTemplate: 100ms + 150ms + 200ms = 450ms total.
  • Parallel calls with WebClient: max(100, 150, 200) = 200ms total!
  • Non-blocking means thread doesn't wait — handles other requests.
  • Use Mono.zip() to combine multiple reactive calls
WebClient Configuration
Java
@Configuration
public class WebClientConfig {

    @Bean
    public WebClient webClient(WebClient.Builder builder) {
        return builder
            // Base URL
            .baseUrl("https://api.external-service.com")

            // Default headers
            .defaultHeader(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE)
            .defaultHeader("X-API-Version", "v1")

            // Timeouts
            .clientConnector(new ReactorClientHttpConnector(
                HttpClient.create()
                    .responseTimeout(Duration.ofSeconds(10))
                    .option(ChannelOption.CONNECT_TIMEOUT_MILLIS, 5000)
            ))

            // Filters (interceptors)
            .filter(logRequest())
            .filter(logResponse())

            .build();
    }

    // Advanced configuration with connection pooling
    @Bean
    public WebClient customWebClient() {
        // Connection pool configuration
        ConnectionProvider provider = ConnectionProvider.builder("custom")
            .maxConnections(100)
            .maxIdleTime(Duration.ofSeconds(20))
            .maxLifeTime(Duration.ofSeconds(60))
            .pendingAcquireTimeout(Duration.ofSeconds(45))
            .evictInBackground(Duration.ofSeconds(120))
            .build();

        HttpClient httpClient = HttpClient.create(provider)
            .responseTimeout(Duration.ofSeconds(10))
            .doOnConnected(conn ->
                conn.addHandlerLast(new ReadTimeoutHandler(10))
                    .addHandlerLast(new WriteTimeoutHandler(10))
            );

        return WebClient.builder()
            .clientConnector(new ReactorClientHttpConnector(httpClient))
            .build();
    }

    // Request logging filter
    private ExchangeFilterFunction logRequest() {
        return ExchangeFilterFunction.ofRequestProcessor(clientRequest -> {
            System.out.println("Request: " + clientRequest.method() + " " + clientRequest.url());
            clientRequest.headers().forEach((name, values) ->
                values.forEach(value -> System.out.println(name + ": " + value))
            );
            return Mono.just(clientRequest);
        });
    }

    // Response logging filter
    private ExchangeFilterFunction logResponse() {
        return ExchangeFilterFunction.ofResponseProcessor(clientResponse -> {
            System.out.println("Response: " + clientResponse.statusCode());
            return Mono.just(clientResponse);
        });
    }
}
WebClient Usage Examples
Java
@Service
@RequiredArgsConstructor
public class UserServiceClient {

    private final WebClient webClient;

    // ===== GET REQUEST - Returns Mono (single value) =====
    public Mono<User> getUserById(Long id) {
        return webClient
            .get()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class);
    }

    // GET - Blocking call (convert to sync)
    public User getUserByIdSync(Long id) {
        return webClient
            .get()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class)
            .block();  // Convert to blocking call
    }

    // ===== GET REQUEST - Returns Flux (multiple values) =====
    public Flux<User> getAllUsers() {
        return webClient
            .get()
            .uri("/users")
            .retrieve()
            .bodyToFlux(User.class);
    }

    // GET with query parameters
    public Flux<User> searchUsers(String name, Integer age) {
        return webClient
            .get()
            .uri(uriBuilder -> uriBuilder
                .path("/users/search")
                .queryParam("name", name)
                .queryParam("age", age)
                .build())
            .retrieve()
            .bodyToFlux(User.class);
    }

    // ===== POST REQUEST =====
    public Mono<User> createUser(CreateUserRequest request) {
        return webClient
            .post()
            .uri("/users")
            .contentType(MediaType.APPLICATION_JSON)
            .bodyValue(request)  // Or .body(Mono.just(request), CreateUserRequest.class)
            .retrieve()
            .bodyToMono(User.class);
    }

    // POST with custom headers
    public Mono<User> createUserWithHeaders(CreateUserRequest request) {
        return webClient
            .post()
            .uri("/users")
            .header("X-Idempotency-Key", UUID.randomUUID().toString())
            .header(HttpHeaders.AUTHORIZATION, "Bearer " + getToken())
            .bodyValue(request)
            .retrieve()
            .bodyToMono(User.class);
    }

    // ===== PUT REQUEST =====
    public Mono<User> updateUser(Long id, UpdateUserRequest request) {
        return webClient
            .put()
            .uri("/users/{id}", id)
            .bodyValue(request)
            .retrieve()
            .bodyToMono(User.class);
    }

    // ===== DELETE REQUEST =====
    public Mono<Void> deleteUser(Long id) {
        return webClient
            .delete()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(Void.class);
    }

    // ===== RETRIEVE WITH STATUS & HEADERS =====
    public Mono<ResponseEntity<User>> getUserWithDetails(Long id) {
        return webClient
            .get()
            .uri("/users/{id}", id)
            .retrieve()
            .toEntity(User.class);  // Returns ResponseEntity with headers, status
    }

    // ===== ERROR HANDLING =====
    public Mono<User> getUserWithErrorHandling(Long id) {
        return webClient
            .get()
            .uri("/users/{id}", id)
            .retrieve()
            .onStatus(
                HttpStatus::is4xxClientError,
                response -> Mono.error(new ClientErrorException("Client error"))
            )
            .onStatus(
                HttpStatus::is5xxServerError,
                response -> Mono.error(new ServerErrorException("Server error"))
            )
            .bodyToMono(User.class)
            .onErrorResume(e -> {
                System.out.println("Error: " + e.getMessage());
                return Mono.empty();  // Return empty on error
            });
    }

    // ===== PARALLEL REQUESTS (Non-blocking!) =====
    public Mono<DashboardData> getDashboard(Long userId) {
        Mono<User> userMono = getUserById(userId);
        Mono<List<Order>> ordersMono = getUserOrders(userId);
        Mono<Analytics> analyticsMono = getUserAnalytics(userId);

        // Combine 3 API calls in parallel!
        return Mono.zip(userMono, ordersMono, analyticsMono)
            .map(tuple -> new DashboardData(
                tuple.getT1(),  // User
                tuple.getT2(),  // Orders
                tuple.getT3()   // Analytics
            ));
    }

    // ===== RETRY LOGIC =====
    public Mono<User> getUserWithRetry(Long id) {
        return webClient
            .get()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class)
            .retryWhen(Retry.backoff(3, Duration.ofSeconds(1))
                .maxBackoff(Duration.ofSeconds(5))
                .filter(throwable -> throwable instanceof WebClientRequestException)
            );
    }

    // ===== TIMEOUT =====
    public Mono<User> getUserWithTimeout(Long id) {
        return webClient
            .get()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class)
            .timeout(Duration.ofSeconds(5))
            .onErrorResume(TimeoutException.class, e ->
                Mono.error(new ServiceTimeoutException("User service timeout"))
            );
    }
}
💡 WebClient Best Practices
  • Use WebClient over RestTemplate for all new projects
  • Configure connection pooling for better performance
  • Set proper timeouts (connection, read, response)
  • Use Mono.zip() for parallel API calls — massive performance gain
  • Add retry logic with exponential backoff for resilience
  • Use filters for cross-cutting concerns (logging, auth, metrics)
  • Don't call .block() in reactive controllers — defeats the purpose

Feign Client
Declarative HTTP Client
Feign is a declarative REST client from Spring Cloud Netflix. Define API calls as Java interfaces with annotations. Feign generates implementation at runtime. Integrates with Eureka (service discovery), Ribbon (load balancing), Hystrix (circuit breaker). Perfect for microservices.
⚡ When to Use Feign Client
  • Microservices architecture with service discovery (Eureka)
  • Multiple service-to-service calls with clean API
  • Prefer declarative style over imperative HTTP calls
  • Need built-in load balancing and circuit breaker
  • Want minimal boilerplate for REST clients
🏢 Real Scenario — Order Service calling Payment Service
  • Order Service needs to charge payment via Payment Service.
  • Define PaymentServiceClient interface with @FeignClient.
  • Add method: ChargeResponse charge(ChargeRequest request).
  • Feign generates implementation, handles service discovery, load balancing.
  • Clean, testable code — mock interface in tests

@SpringBootApplication
@EnableFeignClients  // Enable Feign client scanning
public class OrderServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(OrderServiceApplication.class, args);
    }
}

// ===== FEIGN CLIENT INTERFACE =====
@FeignClient(
    name = "payment-service",        // Service name in Eureka
    url = "${payment.service.url}", // Fallback URL (if not using Eureka)
    configuration = FeignConfig.class  // Custom configuration
)
public interface PaymentServiceClient {

    // GET request
    @GetMapping("/api/v1/payments/{id}")
    PaymentDTO getPayment(@PathVariable("id") Long id);

    // POST request
    @PostMapping("/api/v1/payments")
    PaymentResponse createPayment(@RequestBody PaymentRequest request);

    // GET with query parameters
    @GetMapping("/api/v1/payments")
    List<PaymentDTO> searchPayments(
        @RequestParam("status") String status,
        @RequestParam("from") LocalDate from
    );

    // POST with headers
    @PostMapping("/api/v1/payments/charge")
    ChargeResponse charge(
        @RequestHeader("X-Idempotency-Key") String idempotencyKey,
        @RequestBody ChargeRequest request
    );

    // PUT request
    @PutMapping("/api/v1/payments/{id}")
    PaymentDTO updatePayment(
        @PathVariable("id") Long id,
        @RequestBody UpdatePaymentRequest request
    );

    // DELETE request
    @DeleteMapping("/api/v1/payments/{id}")
    void deletePayment(@PathVariable("id") Long id);
}
Feign Configuration
Java
@Configuration
public class FeignConfig {

    // Logging level
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;  // NONE, BASIC, HEADERS, FULL
    }

    // Custom encoder
    @Bean
    public Encoder feignEncoder() {
        return new SpringEncoder(new HttpMessageConvertersAutoConfiguration()
            .messageConverters());
    }

    // Custom decoder
    @Bean
    public Decoder feignDecoder() {
        return new SpringDecoder(new HttpMessageConvertersAutoConfiguration()
            .messageConverters());
    }

    // Request interceptor (add headers to all requests)
    @Bean
    public RequestInterceptor requestInterceptor() {
        return requestTemplate -> {
            requestTemplate.header("X-API-Version", "v1");
            requestTemplate.header("Authorization", "Bearer " + getToken());
        };
    }

    // Error decoder (custom exception handling)
    @Bean
    public ErrorDecoder errorDecoder() {
        return (methodKey, response) -> {
            if (response.status() == 404) {
                return new ResourceNotFoundException("Resource not found");
            }
            if (response.status() >= 500) {
                return new ServerErrorException("Server error");
            }
            return new Exception("Generic error");
        };
    }

    // Timeouts
    @Bean
    public Request.Options options() {
        return new Request.Options(
            5000,  // connectTimeout (ms)
            10000  // readTimeout (ms)
        );
    }
}

Using Feign Client
Java
@Service
@RequiredArgsConstructor
public class OrderService {

    private final PaymentServiceClient paymentClient;  // Injected automatically
    private final OrderRepository orderRepository;

    public Order placeOrder(OrderRequest request) {
        // Create order
        Order order = new Order();
        order.setCustomerId(request.getCustomerId());
        order.setTotal(request.getTotal());
        orderRepository.save(order);

        // Call payment service via Feign
        PaymentRequest paymentRequest = new PaymentRequest();
        paymentRequest.setOrderId(order.getId());
        paymentRequest.setAmount(request.getTotal());

        try {
            PaymentResponse payment = paymentClient.createPayment(paymentRequest);
            order.setPaymentId(payment.getId());
            order.setStatus("PAID");

        } catch (FeignException e) {
            // Handle Feign errors
            order.setStatus("PAYMENT_FAILED");
            throw new PaymentException("Payment failed", e);
        }

        return orderRepository.save(order);
    }

    public PaymentDTO getOrderPayment(Long orderId) {
        Order order = orderRepository.findById(orderId).orElseThrow();
        return paymentClient.getPayment(order.getPaymentId());
    }
}

📌 Note:
  • Feign requires Spring Cloud dependencies — add BOM to pom.xml
  • Use @EnableFeignClients on main application class
  • Service discovery (Eureka) integration automatic if on classpath
  • Load balancing (Ribbon) integrated by default
  • Circuit breaker (Hystrix/Resilience4j) requires additional config
💡 Feign Best Practices
  • Use Feign for microservices — cleanest API for service-to-service calls
  • Define interfaces in shared library for client/server contract
  • Always configure timeouts — don't rely on defaults
  • Implement fallbacks for graceful degradation
  • Use RequestInterceptor for common headers (auth, tracing)
  • Enable logging in dev/staging for debugging
  • Consider retry logic for transient failures
