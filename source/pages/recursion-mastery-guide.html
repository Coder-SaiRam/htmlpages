FUNDAMENTALS
What is Recursion?
Understanding the core concept and components
RECURSION DEFINITION
  • Recursion: A function that calls itself to solve a smaller version of the same problem
  • Base Case: The terminating condition that stops recursion
  • Recursive Case: The part where function calls itself with modified parameters
  • Call Stack: Memory structure that tracks function calls
BasicRecursion.java
java
// Classic Example: Factorial
public int factorial(int n) {
    // BASE CASE: Stop condition
    if (n == 0 || n == 1) {
        return 1;
    }

    // RECURSIVE CASE: Break down the problem
    return n * factorial(n - 1);
}

// Visualization of factorial(4):
// factorial(4) = 4 * factorial(3)
//              = 4 * (3 * factorial(2))
//              = 4 * (3 * (2 * factorial(1)))
//              = 4 * (3 * (2 * 1))
//              = 24

KEY COMPONENTS OF RECURSION
  • Base Case: MUST have at least one to prevent infinite recursion
  • Progress Toward Base: Each recursive call must move closer to base case
  • Trust the Recursion: Assume the recursive call works for smaller inputs
  • Combine Results: Use the result from recursive call to build solution

CORE CONCEPTS
The Three Rules of Recursion
Essential principles every recursive function must follow
Rule Description Example
1. Base Case Must have a stopping condition if (n == 0) return 1;
2. Move Toward Base Each call must progress toward the base case factorial(n-1) reduces n each time
3. Call Itself Function must call itself recursively return n * factorial(n-1);
COMMON PITFALLS TO AVOID
  • Missing Base Case: Causes StackOverflowError (infinite recursion)
  • Wrong Base Case: May never be reached or return incorrect result
  • Not Progressing: Calling with same parameter (e.g., func(n) instead of func(n-1))
  • Multiple Base Cases: Don't forget edge cases (e.g., negative numbers)
  • Excessive Recursion: Some problems are better solved iteratively
RecursionMistakes.java
java
// ❌ WRONG: Missing base case
public int badFactorial(int n) {
    return n * badFactorial(n - 1); // Infinite recursion!
}

// ❌ WRONG: Not progressing toward base case
public int badFactorial2(int n) {
    if (n == 0) return 1;
    return n * badFactorial2(n); // Same n! Infinite loop!
}

// ✅ CORRECT: Has base case and progresses
public int goodFactorial(int n) {
    if (n <= 1) return 1; // Base case (handles 0 and 1)
    return n * goodFactorial(n - 1); // Progresses toward base
}

TYPES
Types of Recursion
Different recursive structures and their use cases

1. Linear Recursion (Single Recursive Call)

LinearRecursion.java
java
// Linear Recursion: One recursive call per execution
public int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 1); // Single recursive call
}

// Time: O(n), Space: O(n) for call stack

2. Binary Recursion (Two Recursive Calls)

BinaryRecursion.java
java
// Binary Recursion: Two recursive calls per execution
public int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2); // Two recursive calls
}

// Time: O(2^n) - Exponential!, Space: O(n) for deepest call

3. Multiple Recursion (Many Recursive Calls)

MultipleRecursion.java
java
// Multiple Recursion: Many recursive calls (e.g., in loops)
public void printPermutations(String prefix, String remaining) {
    if (remaining.length() == 0) {
        System.out.println(prefix);
        return;
    }

    for (int i = 0; i < remaining.length(); i++) {
        String newPrefix = prefix + remaining.charAt(i);
        String newRemaining = remaining.substring(0, i) +
                             remaining.substring(i + 1);
        printPermutations(newPrefix, newRemaining); // Multiple calls in loop
    }
}

4. Tail Recursion (Optimizable)

TailRecursion.java
java
// Tail Recursion: Recursive call is the last operation
public int factorialTail(int n, int accumulator) {
    if (n == 0) return accumulator;
    return factorialTail(n - 1, n * accumulator); // Last operation
}

// Can be optimized by compiler to iterative form (not in Java though)
// Time: O(n), Space: O(1) with tail call optimization
CHOOSING THE RIGHT TYPE
  • Linear: Simple problems with sequential reduction (sum, reverse)
  • Binary: Problems with two subproblems (fibonacci, binary search tree)
  • Multiple: Problems with many choices (permutations, combinations)
  • Tail: When you want potential optimization (accumulators pattern)

BASIC PATTERNS
Essential Recursion Templates
Common patterns you'll use in 80% of problems

Pattern 1: Process and Recurse

WHEN TO USE
  • Need to perform an action at each step
  • Building result as you go down the recursion
  • Examples: Print elements, count nodes, calculate sum
ProcessAndRecurse.java
java
// Pattern: Process current, then recurse
public void printArray(int[] arr, int index) {
    // Base case
    if (index >= arr.length) return;

    // Process: Print current element
    System.out.print(arr[index] + " ");

    // Recurse: Move to next
    printArray(arr, index + 1);
}

// Count elements
public int countElements(int[] arr, int index) {
    if (index >= arr.length) return 0;
    return 1 + countElements(arr, index + 1);
}

Pattern 2: Recurse and Process

WHEN TO USE
  • Need result from recursive call before processing
  • Building result on the way back up
  • Examples: Reverse operations, bottom-up calculations
RecurseAndProcess.java
java
// Pattern: Recurse first, then process
public void printReverse(int[] arr, int index) {
    // Base case
    if (index >= arr.length) return;

    // Recurse first: Go to end
    printReverse(arr, index + 1);

    // Process: Print on way back (reverse order)
    System.out.print(arr[index] + " ");
}

// Calculate factorial (need result from below)
public int factorial(int n) {
    if (n <= 1) return 1;
    int result = factorial(n - 1); // Recurse first
    return n * result; // Then process
}

Pattern 3: Divide and Conquer

WHEN TO USE
  • Split problem into smaller subproblems
  • Solve each subproblem recursively
  • Combine results from subproblems
  • Examples: Merge sort, binary search, quick sort
DivideAndConquer.java
java
// Pattern: Divide, Conquer, Combine
public int binarySearch(int[] arr, int target, int left, int right) {
    // Base case
    if (left > right) return -1;

    // Divide: Split problem in half
    int mid = left + (right - left) / 2;

    if (arr[mid] == target) return mid;

    // Conquer: Recursively search one half
    if (arr[mid] > target) {
        return binarySearch(arr, target, left, mid - 1);
    } else {
        return binarySearch(arr, target, mid + 1, right);
    }
}

// Find maximum in array
public int findMax(int[] arr, int left, int right) {
    if (left == right) return arr[left];

    int mid = (left + right) / 2;
    int leftMax = findMax(arr, left, mid);
    int rightMax = findMax(arr, mid + 1, right);

    // Combine: Take maximum of both halves
    return Math.max(leftMax, rightMax);
}

WHEN TO USE

private void collectNodes(TreeNode node, List<Integer> result) {
    if (node == null) return;
    result.add(node.val);
    collectNodes(node.left, result);
    collectNodes(node.right, result);
}

PATTERN
The Backtracking Template
Choose → Explore → Unchoose
📌 UNIVERSAL BACKTRACKING TEMPLATE
  • Choose - Make a decision and add it to your current path
  • Explore - Recursively explore further with this choice
  • Unchoose - Remove the choice (backtrack) to try other options
BacktrackTemplate.java
java
public class BacktrackTemplate {
    public void backtrack(State state, List<Result> results) {
        // BASE CASE: Found a valid solution
        if (isValidSolution(state)) {
            results.add(new Result(state));
            return;
        }

        // ITERATE through all possible choices
        for (Choice choice : getPossibleChoices(state)) {
            // 1. CHOOSE: Make a choice
            state.choose(choice);

            // 2. EXPLORE: Recurse with this choice
            backtrack(state, results);

            // 3. UNCHOOSE: Backtrack (undo choice)
            state.unchoose(choice);
        }
    }

    // Example: Generate all permutations
    public List<List<Integer>> permute(int[] nums) {
        List<List<Integer>> results = new ArrayList<>();
        backtrackPermute(nums, new ArrayList<>(), new boolean[nums.length], results);
        return results;
    }

    private void backtrackPermute(int[] nums, List<Integer> current,
                                   boolean[] used, List<List<Integer>> results) {
        // Base case: permutation complete
        if (current.size() == nums.length) {
            results.add(new ArrayList<>(current));
            return;
        }

        for (int i = 0; i < nums.length; i++) {
            if (used[i]) continue;  // Skip used elements

            current.add(nums[i]);        // CHOOSE
            used[i] = true;
            backtrackPermute(nums, current, used, results);  // EXPLORE
            current.remove(current.size() - 1);  // UNCHOOSE
            used[i] = false;
        }
    }
}
⚡ WHEN TO USE BACKTRACKING
  • Finding all possible solutions (permutations, combinations, subsets)
  • Constraint satisfaction problems (N-Queens, Sudoku)
  • Searching decision trees where you need to explore all paths
  • When you need to "try" something and potentially undo it

// Permutations: Order matters, use all elements
public List<List<Integer>> permute(int[] nums) {
    List<List<Integer>> result = new ArrayList<>();
    backtrack(nums, new ArrayList<>(), result);
    return result;
}

private void backtrack(int[] nums, List<Integer> current, List<List<Integer>> result) {
    if (current.size() == nums.length) {
        result.add(new ArrayList<>(current));
        return;
    }

    for (int num : nums) {
        if (current.contains(num)) continue;  // Skip if already used
        current.add(num);                     // Choose
        backtrack(nums, current, result);     // Explore
        current.remove(current.size() - 1);   // Unchoose
    }
}

// With Duplicates: [1,1,2] → [[1,1,2], [1,2,1], [2,1,1]]
public List<List<Integer>> permuteUnique(int[] nums) {
    List<List<Integer>> result = new ArrayList<>();
    Arrays.sort(nums);  // Sort to handle duplicates
    backtrackUnique(nums, new ArrayList<>(), new boolean[nums.length], result);
    return result;
}

private void backtrackUnique(int[] nums, List<Integer> current,
                             boolean[] used, List<List<Integer>> result) {
    if (current.size() == nums.length) {
        result.add(new ArrayList<>(current));
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;
        // Skip duplicates: if current num = previous num and previous not used
        if (i > 0 && nums[i] == nums[i-1] && !used[i-1]) continue;

        current.add(nums[i]);
        used[i] = true;
        backtrackUnique(nums, current, used, result);
        current.remove(current.size() - 1);
        used[i] = false;
    }
}
💡 KEY INSIGHT
  • contains() check is O(n), use boolean[] for O(1) lookup
  • For duplicates: sort first, skip if nums[i] == nums[i-1] && !used[i-1]

Combinations & Subsets Pattern
Order doesn't matter, varying sizes
CombinationsSubsets.java
java
// Subsets: All possible subsets (power set)
// [1,2,3] → [[], [1], [2], [3], [1,2], [1,3], [2,3], [1,2,3]]
public List<List<Integer>> subsets(int[] nums) {
    List<List<Integer>> result = new ArrayList<>();
    backtrackSubsets(nums, 0, new ArrayList<>(), result);
    return result;
}

private void backtrackSubsets(int[] nums, int start,
                              List<Integer> current, List<List<Integer>> result) {
    result.add(new ArrayList<>(current));  // Add at every level

    for (int i = start; i < nums.length; i++) {
        current.add(nums[i]);                      // Choose
        backtrackSubsets(nums, i + 1, current, result);  // Explore (i+1 to avoid reuse)
        current.remove(current.size() - 1);        // Unchoose
    }
}

// Combinations: Choose k elements from n
// C(4, 2) → [[1,2], [1,3], [1,4], [2,3], [2,4], [3,4]]
public List<List<Integer>> combine(int n, int k) {
    List<List<Integer>> result = new ArrayList<>();
    backtrackCombine(n, k, 1, new ArrayList<>(), result);
    return result;
}

private void backtrackCombine(int n, int k, int start,
                              List<Integer> current, List<List<Integer>> result) {
    if (current.size() == k) {  // Found k elements
        result.add(new ArrayList<>(current));
        return;
    }

    for (int i = start; i <= n; i++) {
        current.add(i);
        backtrackCombine(n, k, i + 1, current, result);  // Start from i+1
        current.remove(current.size() - 1);
    }
}

// Combination Sum: Elements can be reused
// [2,3,6,7], target=7 → [[2,2,3], [7]]
public List<List<Integer>> combinationSum(int[] candidates, int target) {
    List<List<Integer>> result = new ArrayList<>();
    Arrays.sort(candidates);
    backtrackComboSum(candidates, target, 0, new ArrayList<>(), result);
    return result;
}

private void backtrackComboSum(int[] candidates, int target, int start,
                               List<Integer> current, List<List<Integer>> result) {
    if (target == 0) {
        result.add(new ArrayList<>(current));
        return;
    }
    if (target < 0) return;  // Exceeded target

    for (int i = start; i < candidates.length; i++) {
        current.add(candidates[i]);
        backtrackComboSum(candidates, target - candidates[i], i, current, result);  // i not i+1 (reuse)
        current.remove(current.size() - 1);
    }
}
🎯 PATTERN DIFFERENCES
  • Permutations: All elements, order matters → use contains() or used[]
  • Combinations: k elements, order doesn't matter → use start index
  • Subsets: All sizes, order doesn't matter → add at every level
  • Combination Sum: Reuse allowed → pass i not i+1

N-Queens Pattern
Constraint satisfaction with validation
NQueens.java
java
public List<List<String>> solveNQueens(int n) {
    List<List<String>> result = new ArrayList<>();
    char[][] board = new char[n][n];
    for (char[] row : board) Arrays.fill(row, '.');
    backtrack(board, 0, result);
    return result;
}

private void backtrack(char[][] board, int row, List<List<String>> result) {
    if (row == board.length) {  // Placed all queens
        result.add(construct(board));
        return;
    }

    for (int col = 0; col < board.length; col++) {
        if (!isValid(board, row, col)) continue;  // Check constraints

        board[row][col] = 'Q';              // Choose
        backtrack(board, row + 1, result);  // Explore next row
        board[row][col] = '.';              // Unchoose
    }
}

private boolean isValid(char[][] board, int row, int col) {
    // Check column
    for (int i = 0; i < row; i++) {
        if (board[i][col] == 'Q') return false;
    }

    // Check upper-left diagonal
    for (int i = row - 1, j = col - 1; i >= 0 && j >= 0; i--, j--) {
        if (board[i][j] == 'Q') return false;
    }

    // Check upper-right diagonal
    for (int i = row - 1, j = col + 1; i >= 0 && j < board.length; i--, j++) {
        if (board[i][j] == 'Q') return false;
    }

    return true;
}

private List<String> construct(char[][] board) {
    List<String> result = new ArrayList<>();
    for (char[] row : board) {
        result.add(new String(row));
    }
    return result;
}
💡 OPTIMIZATION
  • Use sets to track occupied columns and diagonals: Set<Integer> cols, diag1, diag2
  • Diagonal formula: upper-left = row - col, upper-right = row + col
  • This reduces validation from O(n) to O(1)

// Generate all permutations of a string
public List<String> permutations(String s) {
    List<String> result = new ArrayList<>();
    backtrack(s.toCharArray(), 0, result);
    return result;
}

// Method 1: Swap-based (in-place)
private void backtrack(char[] chars, int index, List<String> result) {
    if (index == chars.length) {
        result.add(new String(chars));
        return;
    }

    for (int i = index; i < chars.length; i++) {
        swap(chars, index, i);              // Choose
        backtrack(chars, index + 1, result); // Explore
        swap(chars, index, i);              // Unchoose (backtrack)
    }
}

// Method 2: StringBuilder-based (clearer)
private void backtrackWithSB(String s, StringBuilder current,
                             boolean[] used, List<String> result) {
    if (current.length() == s.length()) {
        result.add(current.toString());
        return;
    }

    for (int i = 0; i < s.length(); i++) {
        if (used[i]) continue;

        current.append(s.charAt(i));
        used[i] = true;
        backtrackWithSB(s, current, used, result);
        current.deleteCharAt(current.length() - 1);
        used[i] = false;
    }
}

// With duplicates: "AAB" → ["AAB", "ABA", "BAA"]
public List<String> permuteUnique(String s) {
    List<String> result = new ArrayList<>();
    char[] chars = s.toCharArray();
    Arrays.sort(chars);  // Sort to group duplicates
    backtrackUnique(chars, new StringBuilder(), new boolean[chars.length], result);
    return result;
}

private void backtrackUnique(char[] chars, StringBuilder current,
                             boolean[] used, List<String> result) {
    if (current.length() == chars.length) {
        result.add(current.toString());
        return;
    }

    for (int i = 0; i < chars.length; i++) {
        if (used[i]) continue;
        // Skip duplicates: if current == previous and previous not used
        if (i > 0 && chars[i] == chars[i-1] && !used[i-1]) continue;

        current.append(chars[i]);
        used[i] = true;
        backtrackUnique(chars, current, used, result);
        current.deleteCharAt(current.length() - 1);
        used[i] = false;
    }
}
💡 STRING MANIPULATION TIPS
  • Use StringBuilder for building strings (mutable, efficient)
  • Swap-based approach is O(1) space but modifies input
  • Always sort first when handling duplicates

PalindromePartitioning.java

// Partition string into all palindromic substrings
// "aab" → [["a","a","b"], ["aa","b"]]
public List<List<String>> partition(String s) {
    List<List<String>> result = new ArrayList<>();
    backtrack(s, 0, new ArrayList<>(), result);
    return result;
}

private void backtrack(String s, int start, List<String> current,
                      List<List<String>> result) {
    if (start == s.length()) {
        result.add(new ArrayList<>(current));
        return;
    }

    for (int end = start + 1; end <= s.length(); end++) {
        String substring = s.substring(start, end);
        if (isPalindrome(substring)) {
            current.add(substring);              // Choose
            backtrack(s, end, current, result);  // Explore
            current.remove(current.size() - 1);  // Unchoose
        }
    }
}

private boolean isPalindrome(String s) {
    int left = 0, right = s.length() - 1;
    while (left < right) {
        if (s.charAt(left++) != s.charAt(right--)) {
            return false;
        }
    }
    return true;
}

// OPTIMIZATION: Pre-compute palindrome table with DP
private boolean[][] computePalindromeTable(String s) {
    int n = s.length();
    boolean[][] dp = new boolean[n][n];

    // Single characters are palindromes
    for (int i = 0; i < n; i++) {
        dp[i][i] = true;
    }

    // Check for length 2
    for (int i = 0; i < n - 1; i++) {
        if (s.charAt(i) == s.charAt(i + 1)) {
            dp[i][i + 1] = true;
        }
    }

    // Check for lengths 3+
    for (int len = 3; len <= n; len++) {
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            if (s.charAt(i) == s.charAt(j) && dp[i + 1][j - 1]) {
                dp[i][j] = true;
            }
        }
    }

    return dp;
}
🎯 PALINDROME PATTERNS
  • Two Pointer: Expand from center or shrink from edges
  • DP Table: Pre-compute O(n²) for O(1) lookup
  • Recursion: isPalindrome(s) = (s[0] == s[n-1]) && isPalindrome(s[1..n-2])

// All substrings: Contiguous sequences
// "abc" → ["a", "ab", "abc", "b", "bc", "c"]
public List<String> allSubstrings(String s) {
    List<String> result = new ArrayList<>();
    for (int i = 0; i < s.length(); i++) {
        for (int j = i + 1; j <= s.length(); j++) {
            result.add(s.substring(i, j));
        }
    }
    return result;  // O(n²) substrings, O(n³) time with substring creation
}

// All subsequences: Pick or skip each character (order preserved)
// "abc" → ["", "a", "b", "c", "ab", "ac", "bc", "abc"]
public List<String> allSubsequences(String s) {
    List<String> result = new ArrayList<>();
    backtrack(s, 0, new StringBuilder(), result);
    return result;
}

private void backtrack(String s, int index, StringBuilder current,
                      List<String> result) {
    if (index == s.length()) {
        result.add(current.toString());
        return;
    }

    // Choice 1: Include current character
    current.append(s.charAt(index));
    backtrack(s, index + 1, current, result);
    current.deleteCharAt(current.length() - 1);

    // Choice 2: Exclude current character
    backtrack(s, index + 1, current, result);
}

// Letter Case Permutations: "a1b2" → ["a1b2","a1B2","A1b2","A1B2"]
public List<String> letterCasePermutation(String s) {
    List<String> result = new ArrayList<>();
    backtrackCase(s.toCharArray(), 0, result);
    return result;
}

private void backtrackCase(char[] chars, int index, List<String> result) {
    if (index == chars.length) {
        result.add(new String(chars));
        return;
    }

    backtrackCase(chars, index + 1, result);  // Keep current case

    if (Character.isLetter(chars[index])) {
        chars[index] ^= (1 << 5);  // Toggle case (XOR trick)
        backtrackCase(chars, index + 1, result);
        chars[index] ^= (1 << 5);  // Restore
    }
}
💡 SUBSTRING VS SUBSEQUENCE
  • Substring: Contiguous, O(n²) count, use nested loops
  • Subsequence: Can skip chars, O(2ⁿ) count, use recursion
  • XOR trick for case toggle: c ^= (1 << 5) or c ^= 32

    for (int i = start; i < nums.length; i++) {

// Partition array into k equal sum subsets
// [4,3,2,3,5,2,1], k=4 → true (can partition into 4 subsets of sum 5)
public boolean canPartitionKSubsets(int[] nums, int k) {
    int sum = 0;
    for (int num : nums) sum += num;
    if (sum % k != 0) return false;  // Can't divide evenly

    int target = sum / k;
    Arrays.sort(nums);
    reverse(nums);  // Start with largest (pruning optimization)

    return backtrack(nums, new int[k], 0, target);
}

private boolean backtrack(int[] nums, int[] buckets, int index, int target) {
    if (index == nums.length) {
        // Check if all buckets equal target
        for (int bucket : buckets) {
            if (bucket != target) return false;
        }
        return true;
    }

    for (int i = 0; i < buckets.length; i++) {
        // Pruning: skip if adding exceeds target
        if (buckets[i] + nums[index] > target) continue;

        buckets[i] += nums[index];  // Choose
        if (backtrack(nums, buckets, index + 1, target)) return true;  // Explore
        buckets[i] -= nums[index];  // Unchoose

        // Pruning: if bucket was empty and failed, skip other empty buckets
        if (buckets[i] == 0) break;
    }

    return false;
}

// Partition Equal Subset Sum: [1,5,11,5] → true ([1,5,5] and [11])
public boolean canPartition(int[] nums) {
    int sum = 0;
    for (int num : nums) sum += num;
    if (sum % 2 != 0) return false;  // Odd sum can't be split equally

    return backtrack(nums, 0, 0, sum / 2);
}

private boolean backtrack(int[] nums, int index, int currentSum, int target) {
    if (currentSum == target) return true;
    if (index == nums.length || currentSum > target) return false;

    // Include current number
    if (backtrack(nums, index + 1, currentSum + nums[index], target)) {
        return true;
    }

    // Exclude current number
    return backtrack(nums, index + 1, currentSum, target);
}
⚠️ PERFORMANCE PITFALLS
  • Sorting and starting with largest elements prunes search space
  • Skip empty buckets after failure (all empty buckets are equivalent)
  • Use DP memoization for overlapping subproblems

private void backtrack(String s, int start, List<String> parts,
                      List<String> result) {
    if (parts.size() == 4) {
        if (start == s.length()) {  // Used all characters
            result.add(String.join(".", parts));
        }
        return;
    }

    // Try 1, 2, or 3 digits for current part
    for (int len = 1; len <= 3 && start + len <= s.length(); len++) {
        String part = s.substring(start, start + len);

        // Validate: no leading zeros (except "0"), value <= 255
        if ((part.startsWith("0") && part.length() > 1) ||
            Integer.parseInt(part) > 255) {
            continue;
        }

        parts.add(part);
        backtrack(s, start + len, parts, result);
        parts.remove(parts.size() - 1);
    }
}

class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int val) { this.val = val; }
}

// PREORDER: Root → Left → Right
public void preorder(TreeNode root) {
    if (root == null) return;
    process(root);              // Process root first
    preorder(root.left);        // Then left subtree
    preorder(root.right);       // Then right subtree
}

// INORDER: Left → Root → Right (gives sorted order for BST)
public void inorder(TreeNode root) {
    if (root == null) return;
    inorder(root.left);         // Left subtree first
    process(root);              // Then process root
    inorder(root.right);        // Then right subtree
}

// POSTORDER: Left → Right → Root
public void postorder(TreeNode root) {
    if (root == null) return;
    postorder(root.left);       // Left subtree first
    postorder(root.right);      // Then right subtree
    process(root);              // Process root last
}

// Max Depth: Height of tree
public int maxDepth(TreeNode root) {
    if (root == null) return 0;
    int leftDepth = maxDepth(root.left);
    int rightDepth = maxDepth(root.right);
    return 1 + Math.max(leftDepth, rightDepth);
}

// Is Balanced: Left and right subtrees differ by at most 1
public boolean isBalanced(TreeNode root) {
    return checkHeight(root) != -1;
}

private int checkHeight(TreeNode node) {
    if (node == null) return 0;

    int leftHeight = checkHeight(node.left);
    if (leftHeight == -1) return -1;  // Left subtree unbalanced

    int rightHeight = checkHeight(node.right);
    if (rightHeight == -1) return -1;  // Right subtree unbalanced

    if (Math.abs(leftHeight - rightHeight) > 1) return -1;  // Current node unbalanced

    return 1 + Math.max(leftHeight, rightHeight);
}
🎯 TRAVERSAL CHEAT SHEET
  • Preorder: Root first - good for copying tree, prefix expressions
  • Inorder: Left→Root→Right - BST gives sorted order
  • Postorder: Children first - good for deletion, postfix expressions

// Path Sum I: Does root-to-leaf path exist with given sum?
public boolean hasPathSum(TreeNode root, int targetSum) {
    if (root == null) return false;
    if (root.left == null && root.right == null) {  // Leaf node
        return targetSum == root.val;
    }
    int remaining = targetSum - root.val;
    return hasPathSum(root.left, remaining) || hasPathSum(root.right, remaining);
}

// Path Sum II: Find all root-to-leaf paths with given sum
public List<List<Integer>> pathSum(TreeNode root, int targetSum) {
    List<List<Integer>> result = new ArrayList<>();
    backtrack(root, targetSum, new ArrayList<>(), result);
    return result;
}

private void backtrack(TreeNode node, int remaining, List<Integer> path,
                      List<List<Integer>> result) {
    if (node == null) return;

    path.add(node.val);  // Choose

    if (node.left == null && node.right == null && remaining == node.val) {
        result.add(new ArrayList<>(path));  // Found valid path
    }

    backtrack(node.left, remaining - node.val, path, result);   // Explore
    backtrack(node.right, remaining - node.val, path, result);

    path.remove(path.size() - 1);  // Unchoose
}

// Path Sum III: Count all paths (not just root-to-leaf) with given sum
public int pathSumIII(TreeNode root, int targetSum) {
    if (root == null) return 0;

    // Paths starting from current node
    int pathsFromRoot = countPaths(root, targetSum);

    // Paths in left and right subtrees
    int pathsInLeft = pathSumIII(root.left, targetSum);
    int pathsInRight = pathSumIII(root.right, targetSum);

    return pathsFromRoot + pathsInLeft + pathsInRight;
}

private int countPaths(TreeNode node, long remaining) {
    if (node == null) return 0;

    int count = (remaining == node.val) ? 1 : 0;
    count += countPaths(node.left, remaining - node.val);
    count += countPaths(node.right, remaining - node.val);

    return count;
}

// Binary Tree Maximum Path Sum (can start/end anywhere)
private int maxSum = Integer.MIN_VALUE;

public int maxPathSum(TreeNode root) {
    maxPathSumHelper(root);
    return maxSum;
}

private int maxPathSumHelper(TreeNode node) {
    if (node == null) return 0;

    // Get max path sum from left and right (ignore negative)
    int left = Math.max(0, maxPathSumHelper(node.left));
    int right = Math.max(0, maxPathSumHelper(node.right));

    // Update global max (path through current node)
    maxSum = Math.max(maxSum, left + right + node.val);

    // Return max single path for parent
    return node.val + Math.max(left, right);
}
💡 PATH SUM PATTERNS
  • Root-to-Leaf: Check for leaf (left==null && right==null)
  • Any Path: Try starting from each node, use prefix sum or recursion
  • Max Path: Update global variable, return single path to parent

// Build tree from Preorder and Inorder
// Preorder: [3,9,20,15,7], Inorder: [9,3,15,20,7]
public TreeNode buildTree(int[] preorder, int[] inorder) {
    Map<Integer, Integer> inorderMap = new HashMap<>();
    for (int i = 0; i < inorder.length; i++) {
        inorderMap.put(inorder[i], i);  // Value → Index mapping
    }
    return build(preorder, 0, preorder.length - 1, inorder, 0, inorder.length - 1, inorderMap);
}

private TreeNode build(int[] preorder, int preStart, int preEnd,
                      int[] inorder, int inStart, int inEnd,
                      Map<Integer, Integer> inorderMap) {
    if (preStart > preEnd || inStart > inEnd) return null;

    int rootVal = preorder[preStart];  // First element in preorder is root
    TreeNode root = new TreeNode(rootVal);

    int rootIndex = inorderMap.get(rootVal);  // Find root in inorder
    int leftSize = rootIndex - inStart;  // Size of left subtree

    // Build left subtree
    root.left = build(preorder, preStart + 1, preStart + leftSize,
                     inorder, inStart, rootIndex - 1, inorderMap);

    // Build right subtree
    root.right = build(preorder, preStart + leftSize + 1, preEnd,
                      inorder, rootIndex + 1, inEnd, inorderMap);

    return root;
}

// Build tree from Inorder and Postorder
// Inorder: [9,3,15,20,7], Postorder: [9,15,7,20,3]
public TreeNode buildTree2(int[] inorder, int[] postorder) {
    Map<Integer, Integer> inorderMap = new HashMap<>();
    for (int i = 0; i < inorder.length; i++) {
        inorderMap.put(inorder[i], i);
    }
    return build2(postorder, 0, postorder.length - 1, inorder, 0, inorder.length - 1, inorderMap);
}

private TreeNode build2(int[] postorder, int postStart, int postEnd,
                       int[] inorder, int inStart, int inEnd,
                       Map<Integer, Integer> inorderMap) {
    if (postStart > postEnd || inStart > inEnd) return null;

    int rootVal = postorder[postEnd];  // Last element in postorder is root
    TreeNode root = new TreeNode(rootVal);

    int rootIndex = inorderMap.get(rootVal);
    int leftSize = rootIndex - inStart;

    root.left = build2(postorder, postStart, postStart + leftSize - 1,
                      inorder, inStart, rootIndex - 1, inorderMap);

    root.right = build2(postorder, postStart + leftSize, postEnd - 1,
                       inorder, rootIndex + 1, inEnd, inorderMap);

    return root;
}
🎯 TREE CONSTRUCTION RULES
  • Preorder + Inorder: First of preorder = root, split inorder at root
  • Postorder + Inorder: Last of postorder = root, split inorder at root
  • Preorder + Postorder: Only possible for full binary trees
  • Use HashMap for O(1) root lookup in inorder array

What is Memoization?
Cache results to avoid redundant calculations
📌 MEMOIZATION = RECURSION + CACHING
  • Store results of expensive function calls
  • Return cached result when same inputs occur again
  • Converts O(2ⁿ) exponential time to O(n) or O(n²) polynomial time
FibonacciComparison.java
java
// WITHOUT MEMOIZATION: O(2^n) - Recalculates same values
public int fibSlow(int n) {
    if (n <= 1) return n;
    return fibSlow(n - 1) + fibSlow(n - 2);
}
// fib(5) calls fib(3) twice, fib(2) three times, etc.

// WITH MEMOIZATION: O(n) - Cache results
public int fibFast(int n) {
    int[] memo = new int[n + 1];
    Arrays.fill(memo, -1);  // -1 indicates not computed
    return fibMemo(n, memo);
}

private int fibMemo(int n, int[] memo) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];  // Return cached result

    memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
    return memo[n];
}

// Recursion Tree Visualization:
//
//         fib(5)
//        /      \
//     fib(4)   fib(3)  ← fib(3) computed twice
//     /   \     /   \
//  fib(3) fib(2) ...
//  /   \
// fib(2) fib(1)  ← fib(2) computed 3 times
//
// With memoization: Compute once, cache, reuse!
⚡ WHEN TO USE MEMOIZATION
  • Overlapping subproblems: Same calculation repeated multiple times
  • Optimal substructure: Solution built from sub-solutions
  • Pure functions: Same input always produces same output

// Climbing Stairs: n steps, can climb 1 or 2 steps at a time
public int climbStairs(int n) {

    Arrays.fill(memo, -1);
    return climb(n, memo);
}

private int climb(int n, int[] memo) {
    if (n <= 2) return n;
    if (memo[n] != -1) return memo[n];

    memo[n] = climb(n - 1, memo) + climb(n - 2, memo);
    return memo[n];
}

// House Robber: Max money without robbing adjacent houses
public int rob(int[] nums) {
    int[] memo = new int[nums.length];
    Arrays.fill(memo, -1);
    return robFrom(nums, 0, memo);
}

private int robFrom(int[] nums, int index, int[] memo) {
    if (index >= nums.length) return 0;
    if (memo[index] != -1) return memo[index];

    // Choice 1: Rob this house, skip next
    int robCurrent = nums[index] + robFrom(nums, index + 2, memo);

    // Choice 2: Skip this house
    int skipCurrent = robFrom(nums, index + 1, memo);

    memo[index] = Math.max(robCurrent, skipCurrent);
    return memo[index];
}

// Decode Ways: "226" → ["BBF", "BZ", "VF"] = 3 ways
public int numDecodings(String s) {
    int[] memo = new int[s.length()];
    Arrays.fill(memo, -1);
    return decode(s, 0, memo);
}

private int decode(String s, int index, int[] memo) {
    if (index == s.length()) return 1;  // Found valid decoding
    if (s.charAt(index) == '0') return 0;  // Invalid
    if (memo[index] != -1) return memo[index];

    int ways = decode(s, index + 1, memo);  // Take 1 digit

    if (index + 1 < s.length()) {
        int twoDigit = Integer.parseInt(s.substring(index, index + 2));
        if (twoDigit <= 26) {
            ways += decode(s, index + 2, memo);  // Take 2 digits
        }
    }

    memo[index] = ways;
    return ways;
}

// Longest Common Subsequence: "abcde" and "ace" → 3 ("ace")
public int longestCommonSubsequence(String text1, String text2) {
    int[][] memo = new int[text1.length()][text2.length()];
    for (int[] row : memo) Arrays.fill(row, -1);
    return lcs(text1, text2, 0, 0, memo);
}

private int lcs(String s1, String s2, int i, int j, int[][] memo) {
    if (i == s1.length() || j == s2.length()) return 0;
    if (memo[i][j] != -1) return memo[i][j];

    if (s1.charAt(i) == s2.charAt(j)) {
        // Characters match: include and move both pointers
        memo[i][j] = 1 + lcs(s1, s2, i + 1, j + 1, memo);
    } else {
        // Don't match: try skipping from either string
        int skip1 = lcs(s1, s2, i + 1, j, memo);
        int skip2 = lcs(s1, s2, i, j + 1, memo);
        memo[i][j] = Math.max(skip1, skip2);
    }

    return memo[i][j];
}

// Edit Distance: Min operations to convert word1 → word2
public int minDistance(String word1, String word2) {
    int[][] memo = new int[word1.length()][word2.length()];
    for (int[] row : memo) Arrays.fill(row, -1);
    return editDist(word1, word2, 0, 0, memo);
}

private int editDist(String s1, String s2, int i, int j, int[][] memo) {
    if (i == s1.length()) return s2.length() - j;  // Insert remaining
    if (j == s2.length()) return s1.length() - i;  // Delete remaining
    if (memo[i][j] != -1) return memo[i][j];

    if (s1.charAt(i) == s2.charAt(j)) {
        memo[i][j] = editDist(s1, s2, i + 1, j + 1, memo);  // No operation
    } else {
        int insert = editDist(s1, s2, i, j + 1, memo);
        int delete = editDist(s1, s2, i + 1, j, memo);
        int replace = editDist(s1, s2, i + 1, j + 1, memo);
        memo[i][j] = 1 + Math.min(insert, Math.min(delete, replace));
    }

    return memo[i][j];
}

// Unique Paths: Grid m×n, can only move right or down
public int uniquePaths(int m, int n) {
    int[][] memo = new int[m][n];
    return paths(0, 0, m, n, memo);
}

private int paths(int row, int col, int m, int n, int[][] memo) {
    if (row == m - 1 && col == n - 1) return 1;  // Reached destination
    if (row >= m || col >= n) return 0;  // Out of bounds
    if (memo[row][col] != 0) return memo[row][col];

    memo[row][col] = paths(row + 1, col, m, n, memo) +  // Move down
                     paths(row, col + 1, m, n, memo);    // Move right

    return memo[row][col];
}
💡 MEMOIZATION TIPS
  • Use int[] for 1 parameter, int[][] for 2, HashMap for complex states
  • Initialize with -1 or use visited array to detect uncomputed states
  • Always check cache before computing: if (memo[i] != -1) return memo[i];

📌 Backtracking Template

backtracking.txt
pseudocode
FUNCTION backtrack(state, results):
    // BASE CASE: Found valid solution
    IF is_valid_solution(state):
        results.add(state.copy())
        RETURN

    // TRY all possible choices
    FOR EACH choice IN get_choices(state):
        // 1. CHOOSE
        state.add(choice)

        // 2. EXPLORE
        backtrack(state, results)

        // 3. UNCHOOSE (backtrack)
        state.remove(choice)

TIME: O(b^d) where b=branching, d=depth  |  SPACE: O(d)

📌 Permutations Template

permutations.txt
pseudocode
FUNCTION permute(nums, current, used, results):
    // BASE: All elements used
    IF current.size() == nums.length:
        results.add(current.copy())
        RETURN

    FOR i = 0 TO nums.length - 1:
        IF used[i]:
            CONTINUE  // Skip if already in current permutation

        current.add(nums[i])
        used[i] = TRUE
        permute(nums, current, used, results)
        current.remove_last()
        used[i] = FALSE

⚠️ For duplicates: Sort first, skip if nums[i]==nums[i-1] && !used[i-1]
TIME: O(n! × n)  |  SPACE: O(n)

📌 Combinations/Subsets Template

combinations.txt
pseudocode
FUNCTION subsets(nums, start, current, results):
    results.add(current.copy())  // ✨ Add at EVERY level

    FOR i = start TO nums.length - 1:
        current.add(nums[i])
        subsets(nums, i + 1, current, results)  // i+1: No reuse
        current.remove_last()

VARIATIONS:
- Combinations (k elements): Add only when current.size() == k
- Combination Sum (reuse allowed): Pass i instead of i+1
- Target sum: Track remaining, prune if exceeded

TIME: O(2^n)  |  SPACE: O(n)

📌 Tree DFS Template

tree-dfs.txt
pseudocode
FUNCTION dfs(node):
    IF node IS NULL:
        RETURN base_value

    // Process current node (optional)
    process(node.val)

    // Recurse on children
    left_result = dfs(node.left)
    right_result = dfs(node.right)

    // Combine results
    RETURN combine(node.val, left_result, right_result)

EXAMPLES:
- Max Depth: RETURN 1 + max(left_result, right_result)
- Sum: RETURN node.val + left_result + right_result
- Exists: RETURN left_result || right_result

TIME: O(n)  |  SPACE: O(h) where h=height

📌 Path Sum Template

path-sum.txt
pseudocode
FUNCTION hasPathSum(node, target):
    IF node IS NULL:
        RETURN FALSE

    // Leaf node: Check if sum matches
    IF node.left IS NULL AND node.right IS NULL:
        RETURN target == node.val

    remaining = target - node.val
    RETURN hasPathSum(node.left, remaining) OR
           hasPathSum(node.right, remaining)

💡 FOR COLLECTING ALL PATHS:
   - Use backtracking with path list
   - Add node before recursion
   - Remove after recursion (backtrack)

TIME: O(n)  |  SPACE: O(h)

📌 Tree Construction Template

FUNCTION buildTree(preorder, inorder):
    // Create map for O(1) inorder lookup
    inorder_map = create_index_map(inorder)

    FUNCTION build(pre_start, pre_end, in_start, in_end):
        IF pre_start > pre_end:
            RETURN NULL

        root_val = preorder[pre_start]  // First in preorder
        root_index = inorder_map[root_val]
        left_size = root_index - in_start

        root = NEW TreeNode(root_val)
        root.left = build(pre_start+1, pre_start+left_size,
                         in_start, root_index-1)
        root.right = build(pre_start+left_size+1, pre_end,
                          root_index+1, in_end)
        RETURN root

    RETURN build(0, len-1, 0, len-1)

⚠️ Postorder: root = postorder[post_end] (last element)
TIME: O(n)  |  SPACE: O(n)

📌 1D Memoization Template

1d-memo.txt
pseudocode
FUNCTION solve(n):
    memo = ARRAY of size n+1, fill with -1

    FUNCTION helper(i):
        // BASE CASE
        IF i meets base condition:
            RETURN base_value

        // CHECK CACHE
        IF memo[i] != -1:
            RETURN memo[i]

        // COMPUTE & CACHE
        memo[i] = compute_from_subproblems(helper(i-1), helper(i-2), ...)
        RETURN memo[i]

    RETURN helper(n)

EXAMPLES: Fibonacci, Climbing Stairs, House Robber
TIME: O(n) - Each state computed once  |  SPACE: O(n)

📌 2D Memoization Template

2d-memo.txt
pseudocode
FUNCTION solve(s1, s2):
    memo = 2D ARRAY [len1][len2], fill with -1

    FUNCTION helper(i, j):
        // BASE CASES
        IF i OR j out of bounds:
            RETURN base_value

        // CHECK CACHE
        IF memo[i][j] != -1:
            RETURN memo[i][j]

        // COMPUTE based on choices
        IF s1[i] == s2[j]:
            memo[i][j] = 1 + helper(i+1, j+1)  // Match
        ELSE:
            memo[i][j] = MAX(helper(i+1, j), helper(i, j+1))  // Skip one

        RETURN memo[i][j]

    RETURN helper(0, 0)

EXAMPLES: LCS, Edit Distance, Unique Paths
TIME: O(m×n)  |  SPACE: O(m×n)

TIPS
Recursion Best Practices
Pro tips for writing clean recursive code
✨ DESIGN PRINCIPLES
  • Trust the recursion: Assume recursive calls work correctly, focus on base case and combination logic
  • Identify subproblems: Ask "How can I make this input smaller?"
  • Define clear base cases: Handle all edge cases explicitly
  • State modification: Pass by value for simplicity, or use backtracking for efficiency
⚠️ COMMON PITFALLS
  • Missing base case: Leads to infinite recursion and stack overflow
  • Wrong base case logic: Off-by-one errors, returning wrong value
  • Not reducing input: Passing same input → infinite recursion
  • Forgetting to backtrack: In permutations/combinations, must undo choices
  • Deep copy vs shallow copy: Use new ArrayList<>(current) when adding to results
  • Exponential without memo: O(2ⁿ) becomes O(n) with caching
💡 OPTIMIZATION TECHNIQUES
  • Memoization: Cache results to avoid recomputation
  • Pruning: Skip branches that can't lead to solution (e.g., sum exceeded)
  • Sorting: Sort input to group duplicates and enable pruning
  • Early termination: Return immediately when solution found
  • Space optimization: Use iterative approach if call stack too deep
🎯 PATTERN RECOGNITION
  • "All possible" → Backtracking (permutations, combinations, subsets)
  • "Optimal" → Memoization/DP (max, min, count)
  • "Tree/Graph traversal" → DFS recursion
  • "Divide input" → Divide and conquer (merge sort, binary search)
  • "Process children first" → Postorder recursion
  • "Overlapping subproblems" → Add memoization
⏱️ TIME COMPLEXITY PATTERNS
  • Linear recursion: T(n) = T(n-1) + O(1) → O(n)
  • Binary recursion: T(n) = 2T(n-1) + O(1) → O(2ⁿ)
  • Divide & conquer: T(n) = 2T(n/2) + O(n) → O(n log n)
  • With memoization: O(2ⁿ) → O(n) or O(n²)
  • Tree recursion: O(n) where n = nodes
