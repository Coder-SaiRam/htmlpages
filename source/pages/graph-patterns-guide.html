Graph Representations

CHOOSING THE RIGHT REPRESENTATION
  • Use Adjacency List for: DFS, BFS, most interview problems
  • Use Adjacency Matrix for: Dense graphs, quick edge lookup O(1)
  • Use Edge List for: Union Find, Kruskal's MST

DFS Pattern - Go Deep First
Master the fundamental DFS traversal technique
WHEN TO USE DFS
  • Finding connected components

  • Path finding (not necessarily shortest)
  • Backtracking problems
  • Topological sorting
  • Checking if graph is bipartite
DFSTemplate.java
java
// DFS Template - Recursive
class Solution {
    void dfs(int node, Map<Integer, List<Integer>> graph, Set<Integer> visited) {
        // Base case: already visited
        if (visited.contains(node)) return;

        // Mark as visited
        visited.add(node);

        // Process current node (optional)
        process(node);

        // Visit all neighbors
        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            dfs(neighbor, graph, visited);
        }
    }
}

// DFS Template - Iterative (using Stack)
void dfsIterative(int start, Map<Integer, List<Integer>> graph) {
    Set<Integer> visited = new HashSet<>();
    Stack<Integer> stack = new Stack<>();

    stack.push(start);

    while (!stack.isEmpty()) {
        int node = stack.pop();

        if (visited.contains(node)) continue;

        visited.add(node);
        process(node);

        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (!visited.contains(neighbor)) {
                stack.push(neighbor);
            }
        }
    }
}

DFS TIME & SPACE COMPLEXITY
  • Time: O(V + E) where V = vertices, E = edges
  • Space: O(V) for visited set + O(H) for recursion stack (H = height)
  • Worst case space: O(V) when graph is a linked list

DFS PATTERN
Connected Components
Count or find all connected components in a graph
PROBLEM CHARACTERISTICS
  • Count number of separate islands/networks
  • Find all nodes in same component
  • Group connected elements
ConnectedComponents.java
java
// Count Connected Components
class Solution {
    public int countComponents(int n, int[][] edges) {
        // Build adjacency list
        Map<Integer, List<Integer>> graph = new HashMap<>();
        for (int[] edge : edges) {
            graph.computeIfAbsent(edge[0], k -> new ArrayList<>()).add(edge[1]);
            graph.computeIfAbsent(edge[1], k -> new ArrayList<>()).add(edge[0]);
        }

        Set<Integer> visited = new HashSet<>();
        int count = 0;

        // Try DFS from each unvisited node
        for (int i = 0; i < n; i++) {
            if (!visited.contains(i)) {
                dfs(i, graph, visited);
                count++; // Found a new component
            }
        }

        return count;
    }

    private void dfs(int node, Map<Integer, List<Integer>> graph, Set<Integer> visited) {
        visited.add(node);
        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (!visited.contains(neighbor)) {
                dfs(neighbor, graph, visited);
            }
        }
    }
}
KEY PATTERN STEPS
  • Step 1: Build graph representation
  • Step 2: Maintain visited set globally
  • Step 3: Loop through all nodes
  • Step 4: If node not visited, start DFS (found new component)
  • Step 5: Increment component counter

DFS PATTERN
Cycle Detection
Detect cycles in directed and undirected graphs
CYCLE DETECTION APPROACHES
  • Undirected Graph: Use parent tracking
  • Directed Graph: Use recursion stack (3-color algorithm)
CycleDetection.java
java
// Cycle Detection in UNDIRECTED Graph
class Solution {
    public boolean hasCycle(Map<Integer, List<Integer>> graph, int n) {
        Set<Integer> visited = new HashSet<>();

        for (int i = 0; i < n; i++) {
            if (!visited.contains(i)) {
                if (dfs(i, -1, graph, visited)) {
                    return true;
                }
            }
        }
        return false;
    }

    private boolean dfs(int node, int parent, Map<Integer, List<Integer>> graph,
                         Set<Integer> visited) {
        visited.add(node);

        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (!visited.contains(neighbor)) {
                if (dfs(neighbor, node, graph, visited)) {
                    return true;
                }
            } else if (neighbor != parent) {
                // Visited neighbor that's not parent = cycle
                return true;
            }
        }
        return false;
    }
}

// Cycle Detection in DIRECTED Graph (3-Color DFS)
class Solution {
    enum State { WHITE, GRAY, BLACK }

    public boolean hasCycle(Map<Integer, List<Integer>> graph, int n) {
        Map<Integer, State> state = new HashMap<>();

        for (int i = 0; i < n; i++) {
            state.put(i, State.WHITE);
        }

        for (int i = 0; i < n; i++) {
            if (state.get(i) == State.WHITE) {
                if (dfs(i, graph, state)) {
                    return true;
                }
            }
        }
        return false;
    }

    private boolean dfs(int node, Map<Integer, List<Integer>> graph,
                         Map<Integer, State> state) {
        state.put(node, State.GRAY); // Currently exploring

        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (state.get(neighbor) == State.GRAY) {
                // Back edge = cycle
                return true;
            }
            if (state.get(neighbor) == State.WHITE) {
                if (dfs(neighbor, graph, state)) {
                    return true;
                }
            }
        }

        state.put(node, State.BLACK); // Finished exploring
        return false;
    }
}
COMMON PITFALLS
  • In undirected graphs, forgetting to track parent leads to false positives
  • In directed graphs, using only visited set won't detect cycles
  • Need recursion stack (GRAY state) to detect back edges
3-COLOR ALGORITHM EXPLAINED
  • WHITE: Not visited yet
  • GRAY: Currently in recursion stack (being explored)
  • BLACK: Completely explored (all neighbors processed)
  • Cycle exists: If we encounter a GRAY node (back edge)

DFS PATTERN
All Paths From Source to Target
Find all possible paths using backtracking
AllPaths.java
java
// Find All Paths from Source to Target
class Solution {
    public List<List<Integer>> allPathsSourceTarget(int[][] graph) {
        List<List<Integer>> result = new ArrayList<>();
        List<Integer> path = new ArrayList<>();

        path.add(0); // Start from node 0
        dfs(0, graph, path, result, graph.length - 1);

        return result;
    }

    private void dfs(int node, int[][] graph, List<Integer> path,
                       List<List<Integer>> result, int target) {
        // Base case: reached target
        if (node == target) {
            result.add(new ArrayList<>(path));
            return;
        }

        // Explore all neighbors
        for (int neighbor : graph[node]) {
            path.add(neighbor);           // Choose
            dfs(neighbor, graph, path, result, target); // Explore
            path.remove(path.size() - 1);  // Backtrack
        }
    }
}

// Generic Path Finding Template with Visited Set
void findPaths(int node, int target, Map<Integer, List<Integer>> graph,
               List<Integer> path, Set<Integer> visited, List<List<Integer>> result) {
    if (node == target) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
        if (!visited.contains(neighbor)) {
            path.add(neighbor);
            visited.add(neighbor);

            findPaths(neighbor, target, graph, path, visited, result);

            path.remove(path.size() - 1);
            visited.remove(neighbor); // CRITICAL: Remove from visited for other paths
        }
    }
}
BACKTRACKING PATTERN
  • Choose: Add node to path
  • Explore: Recursively visit neighbors
  • Unchoose: Remove node from path (backtrack)
  • For graphs with cycles, maintain visited set and remove in backtrack

BREADTH-FIRST SEARCH
BFS Pattern - Level by Level
Master the BFS traversal for shortest paths
WHEN TO USE BFS
  • Finding shortest path in unweighted graphs
  • Level-order traversal
  • Finding minimum steps/moves
  • All nodes at distance K
  • Word ladder problems
  • Grid problems (shortest path in maze)
BFSTemplate.java
java
// BFS Template for Shortest Path
class Solution {
    public int bfs(int start, int target, Map<Integer, List<Integer>> graph) {
        Queue<Integer> queue = new LinkedList<>();
        Set<Integer> visited = new HashSet<>();

        queue.offer(start);
        visited.add(start);
        int distance = 0;

        while (!queue.isEmpty()) {
            int size = queue.size();

            // Process all nodes at current level
            for (int i = 0; i < size; i++) {
                int node = queue.poll();

                // Check if target found
                if (node == target) {
                    return distance;
                }

                // Add all unvisited neighbors to queue
                for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
                    if (!visited.contains(neighbor)) {
                        queue.offer(neighbor);
                        visited.add(neighbor);
                    }
                }
            }

            distance++; // Increment level
        }

        return -1; // Target not reachable
    }
}

BFS VS DFS
  • BFS: Uses Queue (FIFO), finds shortest path, more memory
  • DFS: Uses Stack/Recursion (LIFO), explores deep, less memory
  • For shortest path in unweighted graph → always use BFS

GridBFS.java
java
// Grid BFS Template (4-directional)
class Solution {
    private static final int[][] DIRECTIONS = {{0,1}, {1,0}, {0,-1}, {-1,0}};

    public int shortestPath(int[][] grid, int[] start, int[] target) {
        int rows = grid.length, cols = grid[0].length;
        Queue<int[]> queue = new LinkedList<>();
        boolean[][] visited = new boolean[rows][cols];

        queue.offer(start);
        visited[start[0]][start[1]] = true;
        int steps = 0;

        while (!queue.isEmpty()) {
            int size = queue.size();

            for (int i = 0; i < size; i++) {
                int[] cell = queue.poll();
                int row = cell[0], col = cell[1];

                // Check if target reached
                if (row == target[0] && col == target[1]) {
                    return steps;
                }

                // Explore 4 directions
                for (int[] dir : DIRECTIONS) {
                    int newRow = row + dir[0];
                    int newCol = col + dir[1];

                    // Check bounds and visited
                    if (isValid(newRow, newCol, rows, cols) &&
                        !visited[newRow][newCol] &&
                        grid[newRow][newCol] != 1) { // 1 = obstacle

                        queue.offer(new int[]{newRow, newCol});
                        visited[newRow][newCol] = true;
                    }
                }
            }

            steps++;
        }

        return -1; // No path found
    }

    private boolean isValid(int row, int col, int rows, int cols) {
        return row >= 0 && row < rows && col >= 0 && col < cols;
    }
}

BFS PATTERN
Bidirectional BFS
Optimize BFS by searching from both ends
WHEN TO USE BIDIRECTIONAL BFS
  • Large search space (exponential branching)
  • Both start and end points are known
  • Word ladder, transformation problems
  • Reduces complexity from O(b^d) to O(b^(d/2))
BidirectionalBFS.java
java
// Bidirectional BFS Template
class Solution {
    public int bidirectionalBFS(String start, String end, Set<String> wordList) {
        Set<String> beginSet = new HashSet<>();
        Set<String> endSet = new HashSet<>();
        Set<String> visited = new HashSet<>();

        beginSet.add(start);
        endSet.add(end);
        int steps = 1;

        while (!beginSet.isEmpty() && !endSet.isEmpty()) {
            // Always expand the smaller set (optimization)
            if (beginSet.size() > endSet.size()) {
                Set<String> temp = beginSet;
                beginSet = endSet;
                endSet = temp;
            }

            Set<String> nextLevel = new HashSet<>();

            for (String word : beginSet) {
                // Generate all possible next words
                for (String nextWord : getNeighbors(word, wordList)) {
                    // If other set contains this word, paths meet!
                    if (endSet.contains(nextWord)) {
                        return steps + 1;
                    }

                    if (!visited.contains(nextWord)) {
                        nextLevel.add(nextWord);
                        visited.add(nextWord);
                    }
                }
            }

            beginSet = nextLevel;
            steps++;
        }

        return 0; // No path found
    }

    private List<String> getNeighbors(String word, Set<String> wordList) {
        List<String> neighbors = new ArrayList<>();
        char[] chars = word.toCharArray();

        for (int i = 0; i < chars.length; i++) {
            char old = chars[i];
            for (char c = 'a'; c <= 'z'; c++) {
                chars[i] = c;
                String newWord = new String(chars);
                if (wordList.contains(newWord)) {
                    neighbors.add(newWord);
                }
            }
            chars[i] = old;
        }

        return neighbors;
    }
}

TOPOLOGICAL SORT
Ordering with Dependencies
Linear ordering of vertices in a DAG
WHEN TO USE TOPOLOGICAL SORT
  • Course scheduling with prerequisites
  • Build/compilation order
  • Task dependencies
  • Only works on Directed Acyclic Graphs (DAG)
  • If cycle exists, topological sort is impossible
TopologicalSort.java
java
// Kahn's Algorithm (BFS-based) - Most Common
class Solution {
    public int[] topologicalSort(int numCourses, int[][] prerequisites) {
        // Build graph and calculate in-degrees
        Map<Integer, List<Integer>> graph = new HashMap<>();
        int[] inDegree = new int[numCourses];

        for (int[] pre : prerequisites) {
            int course = pre[0], prereq = pre[1];
            graph.computeIfAbsent(prereq, k -> new ArrayList<>()).add(course);
            inDegree[course]++;
        }

        // Add all nodes with in-degree 0 to queue
        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < numCourses; i++) {
            if (inDegree[i] == 0) {
                queue.offer(i);
            }
        }

        int[] result = new int[numCourses];
        int index = 0;

        // Process nodes with in-degree 0
        while (!queue.isEmpty()) {
            int node = queue.poll();
            result[index++] = node;

            // Reduce in-degree of neighbors
            for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
                inDegree[neighbor]--;
                if (inDegree[neighbor] == 0) {
                    queue.offer(neighbor);
                }
            }
        }

        // If not all nodes processed, cycle exists
        return index == numCourses ? result : new int[0];
    }
}

// DFS-based Topological Sort
class Solution {
    public List<Integer> topologicalSortDFS(Map<Integer, List<Integer>> graph, int n) {
        List<Integer> result = new ArrayList<>();
        Set<Integer> visited = new HashSet<>();
        Set<Integer> recStack = new HashSet<>();

        for (int i = 0; i < n; i++) {
            if (!visited.contains(i)) {
                if (dfs(i, graph, visited, recStack, result)) {
                    return new ArrayList<>(); // Cycle detected
                }
            }
        }

        Collections.reverse(result); // Reverse to get correct order
        return result;
    }

    private boolean dfs(int node, Map<Integer, List<Integer>> graph,
                         Set<Integer> visited, Set<Integer> recStack,
                         List<Integer> result) {
        visited.add(node);
        recStack.add(node);

        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (recStack.contains(neighbor)) return true; // Cycle
            if (!visited.contains(neighbor)) {
                if (dfs(neighbor, graph, visited, recStack, result)) {
                    return true;
                }
            }
        }

        recStack.remove(node);
        result.add(node); // Add after visiting all descendants
        return false;
    }
}

KAHN'S ALGORITHM STEPS
  • Step 1: Calculate in-degree for all vertices
  • Step 2: Add all vertices with in-degree 0 to queue
  • Step 3: While queue not empty:
    • Remove vertex, add to result
    • Decrease in-degree of neighbors
    • If neighbor's in-degree becomes 0, add to queue
  • Step 4: If result has all vertices → success, else cycle exists
COMMON PITFALLS
  • Forgetting to check if all nodes are processed (cycle detection)
  • Confusing edge direction: [a,b] means b → a (b is prerequisite of a)
  • DFS approach requires reversing the result list

UNION FIND (DISJOINT SET)
Efficiently Track Connected Components
Union-Find with path compression and union by rank
WHEN TO USE UNION FIND
  • Dynamically tracking connected components
  • Detecting cycles in undirected graphs
  • Kruskal's MST algorithm
  • Network connectivity
  • Checking if two nodes are in same component
  • Friend circles, accounts merge
UnionFind.java
java
// Union Find (Disjoint Set Union) with Path Compression & Union by Rank
class UnionFind {
    private int[] parent;
    private int[] rank;
    private int components; // Number of disjoint sets

    public UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        components = n;

        // Initially, each node is its own parent
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            rank[i] = 0;
        }
    }

    // Find with Path Compression
    public int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]); // Path compression
        }
        return parent[x];
    }

    // Union by Rank
    public boolean union(int x, int y) {
        int rootX = find(x);
        int rootY = find(y);

        if (rootX == rootY) {
            return false; // Already in same set (cycle!)
        }

        // Attach smaller rank tree under root of higher rank tree
        if (rank[rootX] < rank[rootY]) {
            parent[rootX] = rootY;
        } else if (rank[rootX] > rank[rootY]) {
            parent[rootY] = rootX;
        } else {
            parent[rootY] = rootX;
            rank[rootX]++;
        }

        components--;
        return true;
    }

    // Check if two nodes are connected
    public boolean isConnected(int x, int y) {
        return find(x) == find(y);
    }

    // Get number of connected components
    public int getComponents() {
        return components;
    }
}

// Example: Count Connected Components
public int countComponents(int n, int[][] edges) {
    UnionFind uf = new UnionFind(n);

    for (int[] edge : edges) {
        uf.union(edge[0], edge[1]);
    }

    return uf.getComponents();
}

        (0)(1)(2)(3)(4)(5)  - Each node is its own parent

After union(0,1):
         0───1  2  3  4  5
         ↑      ↑  ↑  ↑  ↑
        (0)    (2)(3)(4)(5)

After union(2,3):
         0───1  2───3  4  5
         ↑      ↑      ↑  ↑
        (0)    (2)    (4)(5)

After union(0,2):
         0───1
         |
         2───3  4  5
         ↑      ↑  ↑
        (0)    (4)(5)

Path Compression: When we find(3), we make 3 point directly to 0:
         0───1
        /|
       2 3    4  5
TIME COMPLEXITY
  • Find: O(α(n)) ≈ O(1) amortized with path compression
  • Union: O(α(n)) ≈ O(1) amortized
  • α(n) is inverse Ackermann function, grows extremely slowly
  • For practical purposes, both operations are near-constant time
COMMON PITFALLS
  • Forgetting path compression in find() - reduces efficiency
  • Not using union by rank - creates unbalanced trees
  • Calling union before find - must find roots first

SHORTEST PATH
Dijkstra's Algorithm
Find shortest path in weighted graphs (non-negative weights)
WHEN TO USE DIJKSTRA'S
  • Weighted graph with non-negative edge weights
  • Finding shortest path from single source to all vertices
  • Network routing, GPS navigation
  • Cheapest flights, minimum cost problems
Dijkstra.java
java
// Dijkstra's Algorithm using Priority Queue
class Solution {
    class Node {
        int vertex;
        int distance;

        Node(int vertex, int distance) {
            this.vertex = vertex;
            this.distance = distance;
        }
    }

    public int dijkstra(Map<Integer, List<Node>> graph, int src, int target, int n) {
        // Min heap: [distance, vertex]
        PriorityQueue<Node> pq = new PriorityQueue<>(
            (a, b) -> a.distance - b.distance
        );

        int[] dist = new int[n];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        pq.offer(new Node(src, 0));

        while (!pq.isEmpty()) {
            Node current = pq.poll();
            int u = current.vertex;
            int currDist = current.distance;

            // If we reached target, return distance
            if (u == target) {
                return currDist;
            }

            // Skip if we've already found a better path
            if (currDist > dist[u]) {
                continue;
            }

            // Relax all edges from u
            for (Node neighbor : graph.getOrDefault(u, Collections.emptyList())) {
                int v = neighbor.vertex;
                int weight = neighbor.distance;
                int newDist = currDist + weight;

                // If found shorter path to v
                if (newDist < dist[v]) {
                    dist[v] = newDist;
                    pq.offer(new Node(v, newDist));
                }
            }
        }

        return dist[target] == Integer.MAX_VALUE ? -1 : dist[target];
    }
}

// Dijkstra with Path Reconstruction
public List<Integer> dijkstraWithPath(Map<Integer, List<Node>> graph,
                                         int src, int target, int n) {
    int[] dist = new int[n];
    int[] parent = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    Arrays.fill(parent, -1);

    PriorityQueue<Node> pq = new PriorityQueue<>((a, b) -> a.distance - b.distance);
    dist[src] = 0;
    pq.offer(new Node(src, 0));

    while (!pq.isEmpty()) {
        Node curr = pq.poll();
        int u = curr.vertex;

        if (curr.distance > dist[u]) continue;

        for (Node neighbor : graph.getOrDefault(u, Collections.emptyList())) {
            int v = neighbor.vertex;
            int newDist = dist[u] + neighbor.distance;

            if (newDist < dist[v]) {
                dist[v] = newDist;
                parent[v] = u; // Track parent for path reconstruction
                pq.offer(new Node(v, newDist));
            }
        }
    }

    // Reconstruct path
    List<Integer> path = new ArrayList<>();
    for (int at = target; at != -1; at = parent[at]) {
        path.add(at);
    }
    Collections.reverse(path);
    return path;
}
DIJKSTRA'S KEY CONCEPTS
  • Greedy approach: Always pick the closest unvisited vertex
  • Relaxation: Update dist[v] if dist[u] + weight(u,v) < dist[v]
  • Time Complexity: O((V + E) log V) with priority queue
  • Space Complexity: O(V)
  • Limitation: Does NOT work with negative weights

SHORTEST PATH
Bellman-Ford Algorithm
Handles negative weights and detects negative cycles
WHEN TO USE BELLMAN-FORD
  • Graph has negative edge weights
  • Need to detect negative weight cycles
  • Single source shortest path
  • Slower than Dijkstra: O(VE) vs O((V+E) log V)
BellmanFord.java
java
// Bellman-Ford Algorithm
class Solution {
    class Edge {
        int from, to, weight;

        Edge(int from, int to, int weight) {
            this.from = from;
            this.to = to;
            this.weight = weight;
        }
    }

    public int[] bellmanFord(List<Edge> edges, int n, int src) {
        int[] dist = new int[n];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // Relax all edges V-1 times
        for (int i = 0; i < n - 1; i++) {
            for (Edge edge : edges) {
                if (dist[edge.from] != Integer.MAX_VALUE &&
                    dist[edge.from] + edge.weight < dist[edge.to]) {
                    dist[edge.to] = dist[edge.from] + edge.weight;
                }
            }
        }

        // Check for negative weight cycles
        for (Edge edge : edges) {
            if (dist[edge.from] != Integer.MAX_VALUE &&
                dist[edge.from] + edge.weight < dist[edge.to]) {
                System.out.println("Negative weight cycle detected!");
                return null;
            }
        }

        return dist;
    }
}

// Optimized: Early termination if no changes
public int[] bellmanFordOptimized(List<Edge> edges, int n, int src) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;

    for (int i = 0; i < n - 1; i++) {
        boolean updated = false;

        for (Edge edge : edges) {
            if (dist[edge.from] != Integer.MAX_VALUE) {
                int newDist = dist[edge.from] + edge.weight;
                if (newDist < dist[edge.to]) {
                    dist[edge.to] = newDist;
                    updated = true;
                }
            }
        }

        // If no updates in this iteration, we're done
        if (!updated) break;
    }

    return dist;
}
BELLMAN-FORD ALGORITHM STEPS
  • Step 1: Initialize dist[src] = 0, all others = ∞
  • Step 2: Relax all edges V-1 times
  • Step 3: Check one more time for negative cycles
  • Why V-1 times? Longest simple path has V-1 edges

ALL-PAIRS SHORTEST PATH
Floyd-Warshall Algorithm
Find shortest paths between all pairs of vertices
WHEN TO USE FLOYD-WARSHALL
  • Need shortest paths between all pairs of vertices
  • Graph is small (n ≤ 400) due to O(V³) complexity
  • Can handle negative weights (but no negative cycles)
  • Dense graphs represented as adjacency matrix
FloydWarshall.java
java
// Floyd-Warshall Algorithm
class Solution {
    public int[][] floydWarshall(int[][] graph) {
        int n = graph.length;
        int[][] dist = new int[n][n];

        // Initialize distance matrix
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                dist[i][j] = graph[i][j];
            }
        }

        // Try all intermediate vertices k
        for (int k = 0; k < n; k++) {
            for (int i = 0; i < n; i++) {
                for (int j = 0; j < n; j++) {
                    // If path through k is shorter
                    if (dist[i][k] != Integer.MAX_VALUE &&
                        dist[k][j] != Integer.MAX_VALUE &&
                        dist[i][k] + dist[k][j] < dist[i][j]) {
                        dist[i][j] = dist[i][k] + dist[k][j];
                    }
                }
            }
        }

        // Check for negative cycles
        for (int i = 0; i < n; i++) {
            if (dist[i][i] < 0) {
                System.out.println("Negative cycle detected!");
                return null;
            }
        }

        return dist;
    }
}
KEY INSIGHT
  • For each pair (i, j), try all possible intermediate vertices k
  • dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
  • Time: O(V³), Space: O(V²)
  • Order of loops matters: k must be outermost loop!

MINIMUM SPANNING TREE
Kruskal's Algorithm
Find MST using Union Find (edge-based)
WHAT IS MST?
  • Subset of edges connecting all vertices
  • No cycles (it's a tree)
  • Minimum total edge weight
  • For graph with V vertices, MST has exactly V-1 edges
Kruskal.java
java
// Kruskal's Algorithm (Union Find)
class Solution {
    class Edge implements Comparable<Edge> {
        int u, v, weight;

        Edge(int u, int v, int weight) {
            this.u = u;
            this.v = v;
            this.weight = weight;
        }

        @Override
        public int compareTo(Edge other) {
            return this.weight - other.weight;
        }
    }

    public int kruskalMST(int n, List<Edge> edges) {
        // Step 1: Sort edges by weight
        Collections.sort(edges);

        // Step 2: Initialize Union Find
        UnionFind uf = new UnionFind(n);

        int mstCost = 0;
        int edgesUsed = 0;
        List<Edge> mstEdges = new ArrayList<>();

        // Step 3: Process edges in sorted order
        for (Edge edge : edges) {
            // If adding this edge doesn't create cycle
            if (uf.union(edge.u, edge.v)) {
                mstCost += edge.weight;
                mstEdges.add(edge);
                edgesUsed++;

                // MST complete when we have V-1 edges
                if (edgesUsed == n - 1) {
                    break;
                }
            }
        }

        // Check if MST is possible
        if (edgesUsed != n - 1) {
            return -1; // Graph not connected
        }

        return mstCost;
    }
}
KRUSKAL'S ALGORITHM STEPS
  • Step 1: Sort all edges by weight (ascending)
  • Step 2: Initialize Union Find structure
  • Step 3: Iterate through sorted edges:
    • If edge connects two different components → add to MST
    • If edge creates cycle → skip it
  • Step 4: Stop when MST has V-1 edges
  • Time: O(E log E) for sorting

MINIMUM SPANNING TREE
Prim's Algorithm
Find MST using Priority Queue (vertex-based)
PRIM'S VS KRUSKAL'S
  • Prim's: Grows one tree, vertex-based, better for dense graphs
  • Kruskal's: Merges trees, edge-based, better for sparse graphs
  • Both produce same MST weight (though edges may differ)
Prim.java
java
// Prim's Algorithm
class Solution {
    class Edge {
        int vertex, weight;

        Edge(int vertex, int weight) {
            this.vertex = vertex;
            this.weight = weight;
        }
    }

    public int primMST(Map<Integer, List<Edge>> graph, int n) {
        // Min heap: [weight, vertex]
        PriorityQueue<Edge> pq = new PriorityQueue<>(
            (a, b) -> a.weight - b.weight
        );

        boolean[] inMST = new boolean[n];
        int mstCost = 0;
        int edgesUsed = 0;

        // Start from vertex 0
        pq.offer(new Edge(0, 0));

        while (!pq.isEmpty() && edgesUsed < n) {
            Edge curr = pq.poll();
            int u = curr.vertex;
            int weight = curr.weight;

            // Skip if already in MST
            if (inMST[u]) continue;

            // Add vertex to MST
            inMST[u] = true;
            mstCost += weight;
            edgesUsed++;

            // Add all edges from u to unvisited vertices
            for (Edge edge : graph.getOrDefault(u, Collections.emptyList())) {
                if (!inMST[edge.vertex]) {
                    pq.offer(edge);
                }
            }
        }

        // Check if all vertices included
        return edgesUsed == n ? mstCost : -1;
    }
}
PRIM'S ALGORITHM STEPS
  • Step 1: Start from any vertex (usually 0)
  • Step 2: Add all edges from current vertex to PQ
  • Step 3: Pick minimum weight edge to unvisited vertex
  • Step 4: Add vertex to MST, repeat until all vertices included
  • Time: O((V + E) log V) with binary heap

  • Articulation Point: Vertex whose removal increases components
  • Used in network reliability analysis
BridgesArticulationPoints.java
java
// Find Bridges (Critical Connections)
class Solution {
    private int time = 0;

    public List<List<Integer>> findBridges(int n, List<List<Integer>> connections) {
        Map<Integer, List<Integer>> graph = new HashMap<>();
        for (List<Integer> conn : connections) {
            graph.computeIfAbsent(conn.get(0), k -> new ArrayList<>()).add(conn.get(1));
            graph.computeIfAbsent(conn.get(1), k -> new ArrayList<>()).add(conn.get(0));
        }

        int[] disc = new int[n];  // Discovery time
        int[] low = new int[n];   // Lowest reachable
        Arrays.fill(disc, -1);
        List<List<Integer>> bridges = new ArrayList<>();

        dfs(0, -1, disc, low, graph, bridges);

        return bridges;
    }

    private void dfs(int u, int parent, int[] disc, int[] low,
                       Map<Integer, List<Integer>> graph,
                       List<List<Integer>> bridges) {
        disc[u] = low[u] = time++;

        for (int v : graph.getOrDefault(u, Collections.emptyList())) {
            if (v == parent) continue; // Skip parent edge

            if (disc[v] == -1) {
                // v not visited
                dfs(v, u, disc, low, graph, bridges);
                low[u] = Math.min(low[u], low[v]);

                // If v can't reach earlier than u, (u,v) is bridge
                if (low[v] > disc[u]) {
                    bridges.add(Arrays.asList(u, v));
                }
            } else {
                // v already visited (back edge)
                low[u] = Math.min(low[u], disc[v]);
            }
        }
    }
}
KEY CONCEPTS
  • disc[u]: Discovery time of vertex u
  • low[u]: Lowest discovery time reachable from u
  • Bridge condition: low[v] > disc[u]
  • Uses Tarjan's algorithm with DFS

Strongly Connected Components (SCC)
Find maximal strongly connected subgraphs
WHAT ARE SCCs?
  • In directed graph: maximal set where every vertex reaches every other
  • Used in: dependency analysis, social networks, circuit design
  • Kosaraju's or Tarjan's algorithm
KosarajuSCC.java
java
// Kosaraju's Algorithm for SCCs
class Solution {
    public List<List<Integer>> findSCCs(Map<Integer, List<Integer>> graph, int n) {
        // Step 1: Fill stack with finish times (DFS on original graph)
        Stack<Integer> stack = new Stack<>();
        boolean[] visited = new boolean[n];

        for (int i = 0; i < n; i++) {
            if (!visited[i]) {
                dfs1(i, graph, visited, stack);
            }
        }

        // Step 2: Create reversed graph
        Map<Integer, List<Integer>> reversed = reverseGraph(graph, n);

        // Step 3: DFS on reversed graph in stack order
        Arrays.fill(visited, false);
        List<List<Integer>> sccs = new ArrayList<>();

        while (!stack.isEmpty()) {
            int node = stack.pop();
            if (!visited[node]) {
                List<Integer> scc = new ArrayList<>();
                dfs2(node, reversed, visited, scc);
                sccs.add(scc);
            }
        }

        return sccs;
    }

    private void dfs1(int node, Map<Integer, List<Integer>> graph,
                        boolean[] visited, Stack<Integer> stack) {
        visited[node] = true;
        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (!visited[neighbor]) {
                dfs1(neighbor, graph, visited, stack);
            }
        }
        stack.push(node); // Push after visiting all descendants
    }

    private void dfs2(int node, Map<Integer, List<Integer>> graph,
                        boolean[] visited, List<Integer> scc) {
        visited[node] = true;
        scc.add(node);
        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (!visited[neighbor]) {
                dfs2(neighbor, graph, visited, scc);
            }
        }
    }

    private Map<Integer, List<Integer>> reverseGraph(
            Map<Integer, List<Integer>> graph, int n) {
        Map<Integer, List<Integer>> reversed = new HashMap<>();
        for (int u : graph.keySet()) {
            for (int v : graph.get(u)) {
                reversed.computeIfAbsent(v, k -> new ArrayList<>()).add(u);
            }
        }
        return reversed;
    }
}
KOSARAJU'S ALGORITHM STEPS
  • Step 1: DFS on original graph, store finish times in stack
  • Step 2: Reverse all edges in graph
  • Step 3: DFS on reversed graph in stack order
  • Each DFS tree in step 3 is one SCC
  • Time: O(V + E)

Bipartite Graph Check
Check if graph can be 2-colored
BIPARTITE GRAPH
  • Vertices can be divided into two independent sets
  • No edges within same set (only between sets)
  • Equivalent to: graph is 2-colorable
  • Equivalent to: no odd-length cycles
BipartiteCheck.java
java
// Check if Graph is Bipartite (BFS)
class Solution {
    public boolean isBipartite(int[][] graph) {
        int n = graph.length;
        int[] colors = new int[n];
        Arrays.fill(colors, -1); // -1 = not colored

        // Check each component
        for (int i = 0; i < n; i++) {
            if (colors[i] == -1) {
                if (!bfs(i, graph, colors)) {
                    return false;
                }
            }
        }

        return true;
    }

    private boolean bfs(int start, int[][] graph, int[] colors) {
        Queue<Integer> queue = new LinkedList<>();
        queue.offer(start);
        colors[start] = 0; // Color with 0

        while (!queue.isEmpty()) {
            int node = queue.poll();

            for (int neighbor : graph[node]) {
                if (colors[neighbor] == -1) {
                    // Not colored yet, color with opposite color
                    colors[neighbor] = 1 - colors[node];
                    queue.offer(neighbor);
                } else if (colors[neighbor] == colors[node]) {
                    // Same color as current node = not bipartite
                    return false;
                }
            }
        }

        return true;
    }
}

// DFS Approach
private boolean dfs(int node, int[][] graph, int[] colors, int color) {
    colors[node] = color;

    for (int neighbor : graph[node]) {
        if (colors[neighbor] == -1) {
            if (!dfs(neighbor, graph, colors, 1 - color)) {
                return false;
            }
        } else if (colors[neighbor] == color) {
            return false;
        }
    }

    return true;
}

ALGORITHM HasCycle(graph, n):
  STEP 1: Initialize color array with WHITE for all nodes
  STEP 2: For each node in graph:
          IF color[node] is WHITE:
              IF DFS_Cycle(node) returns TRUE:
                  Return TRUE (cycle found)
  STEP 3: Return FALSE (no cycle)

  STEP 1: Mark node as GRAY (in current path)
  STEP 2: For each neighbor of node:
          IF neighbor is GRAY:
              Return TRUE (back edge = cycle)
          IF neighbor is WHITE:
              IF DFS_Cycle(neighbor) returns TRUE:
                  Return TRUE
  STEP 3: Mark node as BLACK (finished processing)
  STEP 4: Return FALSE

📌 Basic BFS - Shortest Path in Unweighted Graph

basic-bfs.txt
pseudocode
ALGORITHM BFS(start, target, graph):
  STEP 1: Initialize queue and add start node
  STEP 2: Initialize visited set and add start node
  STEP 3: Initialize distance = 0
  STEP 4: While queue is NOT empty:
          Get size of current level
          For each node in current level:
              Dequeue node
              IF node equals target: Return distance
              For each neighbor of node:
                  IF neighbor NOT in visited:
                      Add neighbor to queue
                      Add neighbor to visited
          Increment distance by 1
  STEP 5: Return -1 (target not reachable)

TIME: O(V + E)  |  SPACE: O(V)

  STEP 5: Initialize steps = 0
  STEP 6: While queue is NOT empty:
          Get size of current level
          For each cell in current level:
              Dequeue [row, col]
              IF [row, col] equals target: Return steps
              For each direction [dr, dc]:
                  Calculate newRow = row + dr
                  Calculate newCol = col + dc
                  IF valid position AND NOT visited AND NOT obstacle:
                      Add [newRow, newCol] to queue
                      Mark [newRow, newCol] as visited
          Increment steps by 1
  STEP 7: Return -1 (target not reachable)

TIME: O(rows × cols)  |  SPACE: O(rows × cols)

📌 Multi-Source BFS (Rotting Oranges Pattern)

multi-source-bfs.txt
pseudocode
ALGORITHM MultiSourceBFS(grid):
  STEP 1: Initialize queue = empty
  STEP 2: Initialize freshCount = 0
  STEP 3: Scan entire grid:
          IF cell is SOURCE:
              Add cell to queue
          IF cell is FRESH:
              Increment freshCount
  STEP 4: Initialize time = 0
  STEP 5: While queue is NOT empty AND freshCount > 0:
          Get size of current level
          For each cell in current level:
              Dequeue cell
              For each neighbor in 4 directions:
                  IF neighbor is FRESH:
                      Mark neighbor as SOURCE
                      Add neighbor to queue
                      Decrement freshCount
          Increment time by 1
  STEP 6: IF freshCount equals 0:
              Return time
          ELSE:
              Return -1 (not all cells reachable)

TIME: O(rows × cols)  |  SPACE: O(rows × cols)

📌 Bidirectional BFS

bidirectional-bfs.txt
pseudocode
ALGORITHM BidirectionalBFS(start, end, graph):
  STEP 1: Initialize beginSet = {start}
  STEP 2: Initialize endSet = {end}
  STEP 3: Initialize visited = empty set
  STEP 4: Initialize steps = 1
  STEP 5: While both sets are NOT empty:
          IF beginSet is larger than endSet:
              Swap beginSet and endSet
          Initialize nextLevel = empty set
          For each node in beginSet:
              For each neighbor of node:
                  IF neighbor in endSet:
                      Return steps (paths meet!)
                  IF neighbor NOT in visited:
                      Add neighbor to nextLevel
                      Add neighbor to visited
          Set beginSet = nextLevel
          Increment steps by 1
  STEP 6: Return -1 (no path exists)

TIME: O(b^(d/2)) instead of O(b^d)  |  b=branching, d=depth

📌 Topological Sort - Kahn's Algorithm (BFS)

kahn-algorithm.txt
pseudocode
ALGORITHM TopologicalSort(graph, n):
  STEP 1: Initialize inDegree array of size n with 0s
  STEP 2: For each node u in graph:
              For each neighbor v of u:
                  Increment inDegree[v] by 1
  STEP 3: Initialize queue = empty
  STEP 4: For each node from 0 to n-1:
              IF inDegree[node] equals 0:
                  Add node to queue
  STEP 5: Initialize result = empty list
  STEP 6: While queue is NOT empty:
              Dequeue node
              Add node to result
              For each neighbor of node:
                  Decrement inDegree[neighbor] by 1
                  IF inDegree[neighbor] equals 0:
                      Add neighbor to queue
  STEP 7: IF size of result equals n:
              Return result (success)
          ELSE:
              Return empty (cycle exists)

TIME: O(V + E)  |  SPACE: O(V)

📌 Topological Sort - DFS Based

topo-dfs.txt
pseudocode
ALGORITHM TopologicalSortDFS(graph, n):
  STEP 1: Initialize visited = empty set
  STEP 2: Initialize stack = empty
  STEP 3: For each node from 0 to n-1:
              IF node NOT in visited:
                  Call DFS(node, visited, stack)
  STEP 4: Initialize result = empty list
  STEP 5: While stack is NOT empty:
              Pop node from stack
              Add node to result
  STEP 6: Return result

ALGORITHM DFS(node, visited, stack):
  STEP 1: Add node to visited
  STEP 2: For each neighbor of node:
              IF neighbor NOT in visited:
                  Call DFS(neighbor, visited, stack)
  STEP 3: Push node to stack (post-order)

TIME: O(V + E)  |  SPACE: O(V)

📌 Union Find (Disjoint Set Union)

union-find.txt
pseudocode
INITIALIZE UnionFind(n):
  STEP 1: Create parent array of size n
  STEP 2: Create rank array of size n with 0s
  STEP 3: For each i from 0 to n-1:
              Set parent[i] = i (each node is its own parent)

ALGORITHM Find(x):
  STEP 1: IF parent[x] NOT equals x:
              Set parent[x] = Find(parent[x]) (Path Compression)
  STEP 2: Return parent[x]

ALGORITHM Union(x, y):
  STEP 1: rootX = Find(x)
  STEP 2: rootY = Find(y)
  STEP 3: IF rootX equals rootY:
              Return FALSE (already in same set)
  STEP 4: (Union by Rank)
          IF rank[rootX] < rank[rootY]:
              Set parent[rootX] = rootY
          ELSE IF rank[rootX] > rank[rootY]:
              Set parent[rootY] = rootX
          ELSE:
              Set parent[rootY] = rootX
              Increment rank[rootX] by 1
  STEP 5: Return TRUE

TIME: O(α(n)) ≈ O(1) per operation

📌 Dijkstra's Algorithm (Non-negative weights)

dijkstra.txt
pseudocode
ALGORITHM Dijkstra(graph, start, target, n):
  STEP 1: Initialize dist array of size n with INFINITY
  STEP 2: Set dist[start] = 0
  STEP 3: Initialize minHeap (priority queue)
  STEP 4: Add [0, start] to minHeap (distance, node)
  STEP 5: While minHeap is NOT empty:
              Extract [currDist, u] with minimum distance
              IF u equals target:
                  Return currDist
              IF currDist > dist[u]:
                  Continue (skip, already processed)
              For each neighbor [v, weight] of u:
                  Calculate newDist = dist[u] + weight
                  IF newDist < dist[v]:
                      Set dist[v] = newDist
                      Add [newDist, v] to minHeap
  STEP 6: IF dist[target] equals INFINITY:
              Return -1 (not reachable)
          ELSE:
              Return dist[target]

TIME: O((V + E) log V)  |  SPACE: O(V)
⚠️ LIMITATION: Does NOT work with negative edge weights

📌 Bellman-Ford Algorithm (Handles negative weights)

bellman-ford.txt
pseudocode
ALGORITHM BellmanFord(edges, n, start):
  STEP 1: Initialize dist array of size n with INFINITY
  STEP 2: Set dist[start] = 0
  STEP 3: Repeat (n-1) times:
              For each edge [u, v, weight] in edges:
                  IF dist[u] NOT equals INFINITY:
                      IF dist[u] + weight < dist[v]:
                          Set dist[v] = dist[u] + weight
  STEP 4: (Check for negative cycle)
          For each edge [u, v, weight] in edges:
              IF dist[u] NOT equals INFINITY:
                  IF dist[u] + weight < dist[v]:
                      Return NULL (negative cycle detected)
  STEP 5: Return dist array

TIME: O(V × E)  |  SPACE: O(V)
💡 WHY n-1 iterations? Longest simple path has at most (n-1) edges

📌 Floyd-Warshall (All pairs shortest path)

floyd-warshall.txt
pseudocode
ALGORITHM FloydWarshall(graph, n):
  STEP 1: Initialize dist as 2D array (n × n)
  STEP 2: For each i from 0 to n-1:
              For each j from 0 to n-1:
                  IF i equals j:
                      Set dist[i][j] = 0
                  ELSE IF edge exists from i to j:
                      Set dist[i][j] = weight(i, j)
                  ELSE:
                      Set dist[i][j] = INFINITY
  STEP 3: For each intermediate vertex k from 0 to n-1:
              For each source vertex i from 0 to n-1:
                  For each destination vertex j from 0 to n-1:
                      IF dist[i][k] + dist[k][j] < dist[i][j]:
                          Set dist[i][j] = dist[i][k] + dist[k][j]
  STEP 4: Return dist (contains shortest paths between all pairs)

TIME: O(V³)  |  SPACE: O(V²)
⚠️ NOTE: k loop must be outermost!

Kruskal's and Prim's algorithms

📌 Kruskal's MST (Edge-based, uses Union Find)

kruskal-mst.txt
pseudocode
ALGORITHM KruskalMST(edges, n):
  STEP 1: Sort all edges by weight in ascending order
  STEP 2: Initialize UnionFind structure with n nodes
  STEP 3: Initialize mstCost = 0
  STEP 4: Initialize edgesUsed = 0
  STEP 5: For each edge [u, v, weight] in sorted edges:
              IF Union(u, v) returns TRUE:
                  (Edge doesn't create cycle)
                  Add weight to mstCost
                  Increment edgesUsed by 1
                  IF edgesUsed equals (n-1):
                      Break (MST complete)
  STEP 6: IF edgesUsed equals (n-1):
              Return mstCost (success)
          ELSE:
              Return -1 (graph not connected)

TIME: O(E log E)  |  SPACE: O(V)
✨ Best for: Sparse graphs

📌 Prim's MST (Vertex-based, uses Priority Queue)

prim-mst.txt
pseudocode
ALGORITHM PrimMST(graph, n):
  STEP 1: Initialize inMST array of size n with FALSE
  STEP 2: Initialize minHeap (priority queue)
  STEP 3: Add [0, 0] to minHeap (weight=0, start vertex=0)
  STEP 4: Initialize mstCost = 0
  STEP 5: Initialize edgesUsed = 0
  STEP 6: While minHeap is NOT empty AND edgesUsed < n:
              Extract [weight, u] with minimum weight
              IF inMST[u] is TRUE:
                  Continue (skip, already in MST)
              Set inMST[u] = TRUE
              Add weight to mstCost
              Increment edgesUsed by 1
              For each neighbor [v, edgeWeight] of u:
                  IF inMST[v] is FALSE:
                      Add [edgeWeight, v] to minHeap
  STEP 7: IF edgesUsed equals n:
              Return mstCost (success)
          ELSE:
              Return -1 (graph not connected)

TIME: O((V + E) log V)  |  SPACE: O(V)
✨ Best for: Dense graphs

INTERVIEW TIPS
Problem-Solving Strategy
Step-by-step approach to tackle graph problems
STEP-BY-STEP APPROACH
  • Step 1: Identify it's a graph problem
    • Look for nodes/vertices and edges/connections
    • Keywords: connected, path, network, dependencies
  • Step 2: Determine graph type
    • Directed or undirected?
    • Weighted or unweighted?
    • Cyclic or acyclic (DAG)?
  • Step 3: Choose representation
    • Adjacency list (most common)
    • Adjacency matrix (for dense graphs)
    • Edge list (for Union Find)
  • Step 4: Select algorithm
    • Shortest path? → BFS (unweighted) or Dijkstra (weighted)
    • All paths? → DFS with backtracking
    • Cycle? → DFS with colors or Union Find
    • Dependencies? → Topological Sort
    • Components? → DFS/BFS or Union Find
  • Step 5: Consider edge cases
    • Disconnected graph (multiple components)
    • Empty graph, single node
    • Self-loops, duplicate edges
    • Negative weights (affects algorithm choice)
TIME COMPLEXITY QUICK REFERENCE
  • DFS/BFS: O(V + E) - visit each vertex and edge once
  • Dijkstra's: O((V + E) log V) - with priority queue
  • Bellman-Ford: O(VE) - relax all edges V-1 times
  • Floyd-Warshall: O(V³) - three nested loops
  • Topological Sort: O(V + E) - DFS or Kahn's
  • Union Find: O(α(n)) ≈ O(1) - per operation
  • Kruskal's MST: O(E log E) - sorting edges
  • Prim's MST: O((V + E) log V) - with heap
COMMON INTERVIEW MISTAKES
  • Not checking if graph is connected (may need to run DFS from multiple sources)
  • Forgetting to mark nodes as visited (infinite loops)
  • In undirected graphs, forgetting to add edges in both directions
  • Using DFS for shortest path in weighted graphs (use Dijkstra)
  • Not handling negative weights (use Bellman-Ford, not Dijkstra)
  • Modifying visited set incorrectly in backtracking problems
  • Confusing parent tracking in cycle detection
  • Not considering time/space trade-offs
OPTIMIZATION TECHNIQUES
  • Early termination: Return immediately when target found
  • Bidirectional BFS: For large search spaces
  • Union Find: For dynamic connectivity queries
  • Memoization: Cache results in path-counting problems
  • Pruning: Skip already visited nodes in BFS/DFS
  • In-place modifications: Use input grid as visited array
