Why Kubernetes? The Problem It Solves
💀
Without Kubernetes
  • App crashes? Someone must restart it manually
  • Traffic spike? Add servers manually
  • Deploy update? Risk of downtime
  • 100s of containers? Pure operational chaos
  • Server dies? All apps on it are gone
K8s
→
🚀
With Kubernetes
  • Auto-restarts crashed containers instantly
  • Auto-scales pods up/down based on load
  • Zero-downtime rolling updates
  • Manages 1000s of containers automatically
  • Reschedules pods to healthy nodes
✈️
Analogy — Airport Control Tower
  • K8s = Control Tower (manages everything)
  • Containers = Airplanes (your apps)
  • Pods = Flight groups
  • Nodes = Runways (machines)

Building Blocks — Container → Pod → Node → Cluster
→
📦
Container
Smallest runnable unit
A packaged app with all its dependencies (code + libraries + runtime). Built from a Docker Image. Runs the same on any machine. Each container is an isolated process on the OS — like a sealed shipping box.
kubectl describe container
→
🫛
Pod
Smallest K8s unit
Wraps 1 or more containers. All containers in a pod share the same IP address and storage volumes. Pods are ephemeral — they die and get recreated. Think of it as a logical host for your containers.
kubectl get pods
→
🖥️
Node
Worker machine
A physical or virtual machine that runs Pods. Each node has a kubelet (agent), kube-proxy (networking), and a Container Runtime (like Docker or containerd). Reports health to the Control Plane continuously.
kubectl get nodes
🌐
Cluster
The full K8s system
A group of Nodes managed together by Kubernetes. Contains 1 Control Plane (brain) + N Worker Nodes (muscle). Acts as a single logical unit. Can span multiple data centers and cloud regions.
kubectl cluster-info

Full Cluster Architecture — Control Plane + Worker Nodes
☸ Kubernetes Cluster
🧠 CONTROL PLANE Master Node
Brain of the cluster — manages state, scheduling, and the API
🔌 kube-apiserver

The front door of the cluster. Every kubectl command, every internal component talks to the API server. Validates and processes all REST API requests. Analogy: Reception desk of an office

🗄️ etcd

Distributed key-value store that saves ALL cluster state — what pods exist, their status, config, secrets. If etcd dies, the cluster loses its memory. Always back this up! Analogy: The cluster's hard drive

📋 kube-scheduler

Watches for newly created pods with no assigned node and decides which node to place them on. Considers CPU, RAM, affinity rules, taints, tolerations. Analogy: Job placement manager

🔄 controller-manager

Runs control loops that watch the cluster state and fix it. If you want 3 replicas and only 2 are running — it creates the 3rd. Handles node failures, service accounts, etc. Analogy: Maintenance crew

☁️ cloud-controller-manager

Links the cluster to cloud-provider APIs (AWS, GCP, Azure). Manages cloud load balancers, storage volumes, and node lifecycle in the cloud. Only runs if you're on a cloud provider.

💻 kubectl (CLI)

Command-line tool you use to talk to the API server. Not part of the control plane itself, but how you interact with it. kubectl apply -f app.yaml sends your config to the API server.

Core K8s Objects — What You Deploy
🚀
Deployment
Stateless Apps
Manages a set of identical Pods (replicas). Declares your desired state — K8s makes it happen. Handles rolling updates (zero downtime) and rollbacks on failure. Always maintains the replica count you specify.
kubectl create deployment nginx --image=nginx --replicas=3
💡 Analogy: Factory manager — always keeps 3 workers on the floor, replaces if one leaves
🔌
Service
Networking
Provides a stable network endpoint for Pods. Pods die and get new IPs — but the Service IP stays the same. Types: ClusterIP (internal), NodePort (external via port), LoadBalancer (cloud LB).
kubectl expose deployment nginx --port=80 --type=LoadBalancer
💡 Analogy: Phone number — stays the same even if the person (pod) moves
⚙️
ConfigMap + Secret
Configuration
ConfigMap: stores non-sensitive config (DB URL, port numbers). Secret: stores sensitive data (passwords, API keys) — base64 encoded. Both are injected into pods as env vars or mounted as files. Decouples config from code.
kubectl create configmap app-config --from-literal=DB_URL=...
💡 Analogy: .env file — app reads settings from it without hardcoding them
💾
PersistentVolume (PV/PVC)
Storage
Pods are ephemeral — storage dies with them. PersistentVolume (PV) = actual storage resource (disk). PVC = pod's claim to use that storage. StorageClass = type of storage (SSD, HDD). Storage survives pod restarts.
kubectl get pvc → shows all storage claims
💡 Analogy: USB drive — plug into any pod, data survives even if pod is deleted
🗂️
Namespace
Isolation
Virtual cluster inside a cluster. Isolates resources by team or environment. Defaults: default (your app), kube-system (K8s internals), kube-public (public info). Apply RBAC and resource quotas per namespace.
kubectl create namespace staging
💡 Analogy: Floors in an office building — Finance on 2nd, Engineering on 3rd
🗃️
StatefulSet
Stateful Apps
Like Deployment but for stateful apps (databases). Pods get stable, ordered names: mysql-0, mysql-1, mysql-2. Ordered startup/shutdown. Each pod gets its own PVC. Pods are not interchangeable — they have identity.
Use for: MySQL, MongoDB, Kafka, Zookeeper
💡 Analogy: Numbered seats on a flight — you always sit in seat-0, not just any seat

Advanced Objects — Ingress, HPA, DaemonSet, RBAC
🚦
Ingress
HTTP Routing
Single entry point for HTTP/HTTPS traffic. Routes based on URL path or hostname to different Services. Handles TLS (HTTPS). Needs an Ingress Controller (like nginx-ingress) to work.
/api → api-service
/web → frontend-service
💡 Receptionist routing callers to right dept
📈
HPA
Auto Scaling
Horizontal Pod Autoscaler — scales pods up/down based on CPU/memory metrics. CPU > 70%? Adds more pods. Traffic drops? Removes them. Set minReplicas and maxReplicas.
minReplicas: 2, maxReplicas: 10
targetCPU: 70%
💡 Hiring more cashiers when queue grows
👹
DaemonSet
Node-level
Ensures ONE pod runs on every node (or selected nodes). Automatically adds pods when new nodes join, removes when nodes leave. Used for monitoring, logging, networking agents.
Use for: Fluentd, Prometheus node-exporter
Calico, Flannel (CNI plugins)
💡 Security guard on every floor of building
🔐
RBAC
Security
Role Based Access Control. Controls who can do what in K8s. Role/ClusterRole: defines permissions. RoleBinding: assigns role to a user/service account. Principle of least privilege.
Role → what actions allowed
RoleBinding → who gets the role
💡 Office ID card — only opens permitted doors

🎯 Interview Questions & Answers
Pod vs Container? Pod is the K8s unit (can hold multiple containers sharing IP+storage). Container is the actual running app process.
Deployment vs StatefulSet? Deployment for stateless apps (interchangeable pods). StatefulSet for stateful apps (pods have stable identity, ordered scaling).
Service types? ClusterIP (internal only), NodePort (external on a fixed port), LoadBalancer (cloud managed LB), ExternalName (DNS alias).
etcd failure? Cluster loses its state — no new scheduling, existing pods keep running. Critical to back up etcd regularly.
liveness vs readiness? Liveness probe: is the container alive? (restart if dead). Readiness probe: is it ready to serve traffic? (remove from Service if not).
How does K8s scale? HPA scales pods horizontally. VPA adjusts CPU/RAM requests. Cluster Autoscaler adds/removes Nodes based on pending pods.
